支付管理模块功能分析                                                                                                                   
                                  
  模块定位                                                                                                                                                                                                  
  支付管理是系统的资金中枢，统一管理所有收款、退款、对账和资金安全检测。与订单模块深度联动，支持线上线下混合收款场景。

  ---
  前端页面（5个）

  1. 支付列表 (PaymentList.tsx)
  - 统计卡片：总支付数、成功数、待处理、总收入、今日收入、转化率
  - 多维筛选：支付状态、方式、日期范围、金额区间
  - 智能搜索：自动识别支付单号/手机号/交易号/订单号
  - 批量操作：批量同步支付状态
  - 行操作：收款、确认、退款、同步状态
  - 可展开行：支付详情 → 订单概要 → 操作时间线

  2. 支付详情 (PaymentDetail.tsx)
  - 右侧抽屉展示支付+订单信息
  - 支持打印支付凭证（HTML 格式化，含品牌标识和时间线）

  3. 创建支付 (PaymentForm.tsx)
  - 后台手动创建支付记录（用于线下收款场景）
  - 按订单号搜索，自动计算未付余额

  4. 退款申请 (RefundModal.tsx)
  - 发起退款，显示可退金额，输入原因和备注

  5. 对账管理 (Reconciliation.tsx + ReconciliationSimple.tsx)
  - 多标签页：对账概览、金额统计、退款统计、收款统计
  - Excel/PDF 导出
  - 差异金额高亮显示

  6. 可疑支付检测 (SuspiciousPayments.tsx)
  - 风险统计卡片（高中低风险数量）
  - 按类型筛选：重复支付、超额支付、系统错误
  - 标记已处理功能

  后端 API（38个端点）

  支付 CRUD（5个）
  POST/GET /payments、GET/PATCH/DELETE /payments/:id

  支付操作（4个）
  POST /payments/:id/confirm（确认收款）、POST /payments/:id/cancel（取消）、POST /payments/:id/sync-status（同步状态）、GET
  /payments/:id/status

  查询统计（8个）
  按手机号/订单号查、统计概览、趋势数据、订单支付历史、余额信息

  退款工作流（10个）
  创建→审批→执行三阶段：POST refund-requests → PUT approve/reject → POST process，每步记录审计日志

  对账（2个）
  手动触发对账 + 定时自动对账（每日凌晨3点差异报告 + 每5分钟微信状态同步）

  可疑支付检测（2个）
  检测列表 + 标记已处理

  ---
  核心业务流程

  线上收款（微信小程序）：
  下单 → 调起微信支付 → 用户付款 → 微信回调 → 更新支付+订单状态 → 发送通知

  线下收款（后台）：
  创建支付记录 → 确认收款（现金/转账） → 更新订单已付金额 → 状态转换

  退款工作流：
  提交申请 → 审批通过 → 执行退款（原路退回/线下退款） → 更新订单状态 → 释放预约时段

  自动对账（兜底）：
  每5分钟查待支付订单 → 询问微信侧状态 → 发现已支付则自动同步 → 防止回调丢失导致的状态不一致

  ---
  关键设计特点

  1. 幂等处理 — 微信回调使用分布式锁 + 缓存防止重复处理
  2. 状态机驱动 — 支付状态转换受状态机控制，避免非法转换
  3. 双轨对账 — 高频增量对账（5分钟级）+ 每日全量差异报告
  4. 可疑检测 — 自动识别重复支付、超额支付、数据不一致
  5. 混合支付 — 支持一笔订单多次收款（定金+尾款），自动计算余额
