# 乖宝宝儿童影楼管理系统 — 项目开发方案

## 一、项目概述

**乖宝宝儿童影楼管理系统** 是一套完整的摄影工作室数字化管理解决方案，包含三个子系统：

| 子系统 | 技术栈 | 用途 |
|--------|--------|------|
| **后端 API** (`server/baby-photo-backend`) | NestJS 11 + TypeScript + Prisma 6 + PostgreSQL 17 | 业务逻辑与数据服务 |
| **管理后台** (`client/admin-frontend`) | React 18 + TypeScript + Vite 5 + Ant Design 5 | 门店运营管理 |
| **微信小程序** (`client/psyd`) | 原生 WeChat Mini Program + Vant Weapp | 用户端预约/下单/购买 |

**库存域数据流：**
```
采购入库 → stock_transaction (INBOUND) → product.stock_quantity += quantity
                                        → stock_alert (检查是否超上限)
销售出库 → stock_outbound → stock_transaction (OUTBOUND) → product.stock_quantity -= quantity
                                                         → stock_alert (检查是否低于阈值)
库存盘点 → stock_check → stock_check_item → stock_transaction (CHECK_ADJUST)
                                           → product.stock_quantity = 实际值
库存调拨 → stock_transfer → stock_transaction (TRANSFER: 源仓- / 目标仓+)
```

### 4.3 API 设计规范

**统一响应格式：**
```json
{
  "code": 200,
  "message": "操作成功",
  "data": { ... }
}
```

**认证机制：**
- 管理端：JWT token 从标准 `admin_login` 接口获取，通过 `Authorization: Bearer <token>` 传递
- 微信端：独立 JWT token 体系，从 `wx-auth/login` 接口获取，通过 `WxJwtAuthGuard` 守卫
- 拦截器自动处理 401 → 跳转登录页

**分页请求格式：**
```
GET /resource?page=1&pageSize=20
```

**分页响应格式：**
```json
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "items": [...],
    "pagination": {
      "current": 1,
      "pageSize": 20,
      "total": 100
    }
  }
}
```

**API 路由设计原则：**
- `GET` — 查询列表/详情
- `POST` — 创建资源
- `PATCH` — 部分更新
- `PUT` — 全量更新
- `DELETE` — 删除资源
- `POST /:id/action` — 资源操作（取消、确认、审批等）
- `POST /batch` — 批量操作

- `request` 对象：返回 `AxiosResponse<ApiResponse<T>>`，保留完整响应头
- `simple` 对象：自动解包返回 `data` 字段，用于大多数业务调用
- 拦截器自动注入 `admin_token`、处理 401 跳转、统一错误提示

### 6.2 库存管理闭环

```
采购入库流程:
创建采购订单 → 审批 → 发货 → 在途追踪
    → 到货质检 → 确认入库
        → StockTransaction(INBOUND)
            → Product.stock_quantity += quantity
                → StockAlert(检查是否超上限)

销售出库流程:
创建销售订单 → (确认) → 创建出库单 → 审批
    → StockTransaction(OUTBOUND)
        → Product.stock_quantity -= quantity
            → StockAlert(检查是否低于阈值)

库存预警自动检测:
每小时定时任务 (已实现但@Cron被注释)
  → 遍历所有 is_track_stock=true 的商品
    → stock_quantity <= low_stock → 创建 LOW_STOCK 告警
    → stock_quantity >= max_stock → 创建 OVER_STOCK 告警
```

### 7.1 Docker 容器化

```
nginx (反向代理 + SSL)
  ├── → 管理后台 (Vite静态文件 → nginx serves)
```

## 八、开发规范

### 8.1 后端规范

| 规范 | 要求 |
|------|------|
| 模块结构 | `module/controller/service/dto` 四层 |
| DTO 校验 | 使用 `class-validator` 装饰器，所有入参必须有校验 |
| 响应格式 | 统一 `{ code, message, data }` 结构 |
| 错误处理 | 使用 `NestJS Exception Filters`，不抛出原始异常 |
| 数据库操作 | 全部通过 Prisma Service，禁止裸 SQL |
| 命名规范 | `camelCase` 字段 + `snake_case` 数据库列 (@map) |
| 定时任务 | 使用 `@nestjs/schedule` 的 `@Cron` 装饰器 |
| 日志 | 使用 NestJS Logger，关键操作必须有日志 |

### 8.2 前端规范

| 规范 | 要求 |
|------|------|
| 组件类型 | 页面组件放 `pages/`，通用组件放 `components/` |
| API 调用 | 通过 `services/` 层封装，组件不直接调用 Axios |
| 状态管理 | 跨组件共享用 Redux，局部状态用 useState/useReducer |
| 样式 | 使用 Ant Design 内置样式 + 全局 CSS 变量 |
| 国际化 | 当前使用中文硬编码，计划迁移至 i18n |
| 路由 | 使用 `react-router-dom v6` 嵌套路由 |
