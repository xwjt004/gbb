# 团购组团失败支付处理分析

## 1. 团购基本规则

- 团购活动的价格本质是**优惠促销价格**，客户参团时已按团购价付款
- 团购活动有效期为 **48小时**（`GROUP_BUY_EXPIRY_HOURS = 48`，硬编码在 `group-buy.service.ts`）
- 成团最低人数默认 **3人**（`minCount`，可在开团时调整 2-10 人）
- 活动状态：`ACTIVE`（进行中）→ `SUCCESS`（成团成功）或 `FAILED`（组团失败/已取消）

## 2. 支付流程

用户参团后，通过微信支付下单付款。订单会记录 `groupBuyActivityId` 关联到团购活动。支付的金额存入系统的支付记录，状态为 `FULLY_PAID`。

**重要**：客户支付的已经是团购优惠价格，而非原价。

## 3. 48小时后组团失败的处理

### 3.1 核心业务规则

**不退款，按团购价继续接单完成交易**。团购价是客户参团时的优惠价格，无论最终是否成团，订单按此价格正常履行。

> 类比：拼团本质是一种营销锁客手段，客户参团即锁定优惠价格。组团失败仅代表未达到预设的营销人数目标，但不影响已下单客户的权益。

### 3.2 当前代码现状（需修改）

当前 `expireStaleActivities()` 方法的实现与业务规则不符——它试图对已付款订单执行退款。需要修改为：

```
1. 查找所有状态为 ACTIVE 且 expiredAt 已过期的活动
2. 将这些活动的状态批量更新为 FAILED
3. 查找这些活动的参与者记录
4. 将参与者的状态更新为 SUCCESS（表示订单按团购价正常履行）
5. ⚠️ 不做任何退款操作
```

### 3.3 修改要点

| 当前代码行为 | 目标行为 |
|-------------|---------|
| 活动标记为 FAILED 后自动退款 | 活动标记为 FAILED，订单维持原状 |
| 参与者标记为 REFUNDED | 参与者标记为 SUCCESS（按团购价成交） |
| 调用 `processRefund()` 修改支付记录 | 不触碰任何支付记录 |
| 回收 GROUP_BUY 类型优惠券 | 优惠券维持原状 |

### 3.4 组团失败后状态

| 实体 | 理想状态 |
|------|---------|
| GroupBuyActivity | FAILED（团购活动标记为失败） |
| Order | 维持原状，按团购价正常履行 |
| Payment | 维持 FULLY_PAID，不做退款 |
| GroupBuyParticipant | SUCCESS（按团购价成交完成） |

## 4. 需修改的代码

### 4.1 核心文件

**`server/baby-photo-backend/src/modules/group-buy/group-buy.service.ts`** — `expireStaleActivities()` 方法（约第564行）

当前逻辑：
```typescript
async expireStaleActivities() {
  const expired = await this.prisma.groupBuyActivity.findMany({
    where: { status: 'ACTIVE', expiredAt: { lte: new Date() } },
  });
  // 标记活动为 FAILED
  await this.prisma.groupBuyActivity.updateMany({
    where: { id: { in: expiredIds } },
    data: { status: 'FAILED' },
  });
  // 查找已付款支付记录，执行退款 ← 需要移除
  const paidPayments = await this.prisma.payment.findMany({...});
  for (const payment of paidPayments) {
    await this.paymentsService.processRefund(payment.id, '团购过期自动退款'); ← 需要移除
  }
}
```

应改为：
```typescript
async expireStaleActivities() {
  const expired = await this.prisma.groupBuyActivity.findMany({
    where: { status: 'ACTIVE', expiredAt: { lte: new Date() } },
    include: { participants: true },
  });
  for (const activity of expired) {
    // 标记活动为 FAILED
    await this.prisma.groupBuyActivity.update({
      where: { id: activity.id },
      data: { status: 'FAILED' },
    });
    // 参与者标记为 SUCCESS（按团购价成交，不退款）
    await this.prisma.groupBuyParticipant.updateMany({
      where: { activityId: activity.id, status: 'JOINED' },
      data: { status: 'SUCCESS' },
    });
  }
}
```

### 4.2 管理员取消

管理员手动取消活动（`PATCH /group-buy/admin/:id/cancel`）同理：仅标记活动为 `FAILED`，**不做退款处理**，已付款订单正常履行。

### 4.3 参与者退团

参与者退团时，如果已付款但不想继续参团，应**走正常的订单退款流程**（管理员手动处理），而非在退团时自动触发。
