# Phase 3: 业务深度与智能化

## 目标

在 Phase 2 功能增强基础上，深入核心业务场景，引入智能辅助能力。

---

## 3.1 智能预约调度

**现状**: 时间槽管理基础功能已完备，缺少智能调度。

- [ ] 预约冲突检测与建议（相同时间段容量预警）
- [ ] 智能推荐时间槽（根据历史订单高峰/低谷，推荐最优排班）
- [ ] 预约提醒（提前一天短信/微信提醒客户）
- [ ] 拍摄日程看板（日/周/月视图，摄影师任务分配）
- [ ] 预约签到/缺席标记 + 自动释放时间槽

---

## 3.2 套餐灵活定价 ✅

**现状**: 套餐价格为固定值，缺少促销、组合折扣等策略。

- [x] 套餐价格日历（SeasonalPrice 模型 + CRUD + 定价引擎，seasonal → promotion → base 瀑布）
- [x] 限时促销活动（promotionPrice + promotionStart/End 字段 + 前端表单）
- [x] 套餐升级/降级差价计算（POST /orders/:id/change-package + 前端按钮弹窗）
- [x] 推荐搭配（CrossSellPackage 模型 + 订单亲和度分析回退，GET /packages/:id/recommendations）
- [x] 团购/拼团套餐支持（groupMinCount + groupPrice 字段 + 前端展示）
- [x] 优惠券核销集成（CreateOrderDto 支持 couponId + POST /orders/:id/apply-coupon 端点）
- [x] 优惠券管理后台（CouponsModule CRUD + CouponList/CouponForm 前端页面）

---

## 3.3 客户关系管理（CRM）

**现状**: 用户信息分散在 users/wx_users 表，缺少客户运营视角。

- [x] 客户画像页面（消费记录、会员等级、成长值进度）
- [x] 会员等级体系（MemberLevel 模型 + 自动升级定时任务 + 前端配置页）
- [x] 客户流失预警（90天未下单自动标记，每日 3AM 检测）
- [x] 生日/纪念日自动关怀（每日 8AM 检测未来7天生日的客户，创建系统通知）
- [x] 客诉跟进记录（Complaint 模型 + CRUD + 状态流转）

---

## 3.4 移动端适配

**现状**: 管理后台仅支持桌面端。

- [ ] 核心页面响应式适配（订单审批、库存预警、数据概览）
- [ ] 移动端快捷操作面板（拍照确认收款、扫码入库）
- [ ] PWA 支持（离线缓存、桌面快捷方式）
- [ ] 微信小程序管理端（轻量版管理后台）

---

## 3.5 财务对账升级

**现状**: 基础对账功能已存在，缺少自动化和风控。

- [ ] 多平台对账（微信支付 + 支付宝 + 银行转账统一对账）
- [ ] 自动差异标记 + 建议调整分录
- [ ] 财务报表（日/月/季/年报，支持自定义维度）
- [ ] 发票管理（开票申请 → 审核 → 开票 → 邮寄追踪）

---

## 优先级建议

| 优先级 | 项目 | 工作量 | 说明 |
|--------|------|--------|------|
| P0 | CRM 客户管理 | 大 | 核心业务增值点 |
| P1 | 智能预约调度 | 中 | 直接提升门店运营效率 |
| P2 | 套餐灵活定价 | 中 | 营销工具，拉动营收 |
| P2 | 财务对账升级 | 大 | 财务合规需求 |
| P3 | 移动端适配 | 大 | 拓宽使用场景 |
