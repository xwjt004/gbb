# Phase 5: 生态集成与平台化

## 目标

从内部管理系统拓展为开放平台，对接外部生态，实现多门店/多渠道管理。

---

## 5.1 多门店支持

**现状**: 系统设计为单门店架构。

- [ ] Prisma：新增 `stores` 表 + 核心表增加 `store_id` 字段
- [ ] 数据隔离：门店级数据权限（员工只能看本门店数据）
- [ ] 总店管理后台：跨门店数据汇总、对比分析
- [ ] 门店独立配置（营业时间、价格策略、打印机、门店信息）
- [ ] 门店间调拨（库存、客户、订单跨门店流转）

---

## 5.2 开放 API 平台

**现状**: API 仅对内服务，无对外接口文档和鉴权。

- [ ] API 网关（统一鉴权、限流、日志、版本管理）
- [ ] 第三方应用接入管理（AppId/AppSecret + 权限授权）
- [ ] Swagger 文档增强为开发者门户（在线调试、SDK 生成）
- [ ] Webhook 事件通知（订单创建/支付成功/退款完成等事件推送）
- [ ] API 调用统计与配额管理

---

## 5.3 第三方平台对接

**现状**: 系统封闭运行，未对接外部平台。

- [ ] 美团/大众点评订单自动同步（团购验券、评价回复）
- [ ] 抖音生活服务对接（团购套餐上架、核销、结算对账）
- [ ] 小红书电商对接（商品同步、订单处理）
- [ ] 对接财务软件（用友/金蝶，凭证自动生成）
- [ ] 对接短信服务商（阿里云/腾讯云 SMS）

---

## 5.4 影像资产管理

**现状**: 照片以文件形式存储，无统一管理。

- [ ] 影像中心（按订单/客户/拍摄日期组织照片）
- [ ] 在线选片系统（客户通过小程序在线选片、加选）
- [ ] 修片进度追踪（原片→初修→精修→客户确认 流程管理）
- [ ] AI 智能修图接入（对接第三方 API，自动调色/美颜）
- [ ] 照片交付管理（云盘链接/小程序下载/U盘寄送）

---

## 5.5 连锁加盟支持

**现状**: 单店模式，无法支撑加盟扩张。

- [ ] 加盟商管理（加盟申请→签约→开店→赋能）
- [ ] 总店统一管控（品牌素材、定价策略、营销活动统一下发）
- [ ] 加盟商独立经营（独立后台、独立收款、独立库存）
- [ ] 分账系统（总店抽佣、按比例分润、自动结算）
- [ ] 标准化 SOP 管理（拍摄流程、服务标准、质检模板）

---

## 优先级建议

| 优先级 | 项目 | 工作量 | 说明 |
|--------|------|--------|------|
| P0 | 开放 API 平台 | 大 | 生态基础，支撑所有外部对接 |
| P1 | 第三方平台对接 | 大 | 对接美团/抖音直接带来订单 |
| P2 | 多门店支持 | 极大 | 架构级改造，需谨慎评估 |
| P2 | 影像资产管理 | 大 | 核心业务数字化最后一环 |
| P3 | 连锁加盟支持 | 极大 | 业务扩张阶段的顶层能力 |
