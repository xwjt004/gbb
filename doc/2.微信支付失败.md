# 1.支付失败: 支付结果-支付失败，requestPayment:fail access denide,appid=wxaf0f01abb6462633
  这个问题不是代码问题，是微信支付配置问题。                                          
                                                   
  错误原因： 微信小程序 SDK 调用 wx.requestPayment 时返回 access denied，意思是商户号没有授权给这个小程序使用微信支付。
                                                  
  后端日志确认预支付订单创建成功（拿到了 prepay_id），说明签名、密钥等配置都没问题。问题出在商户号和小程序的绑定关系上。

  需要你在微信商户平台操作以下步骤：                                                                                         
  步骤 1 — 商户平台发起授权                                                                                                 1. 打开 https://pay.weixin.qq.com/ → 用商户号 1113511352 登录                       
  2. 进入 产品中心 → 产品授权 → AppID 授权管理
  3. 添加 AppID：wxaf0f01abb6462633
  4. 提交后等待小程序管理员确认

  步骤 2 — 小程序平台确认授权
  1. 打开 https://mp.weixin.qq.com/ → 用小程序账号登录
  2. 进入 功能 → 微信支付 → 商户号管理
  3. 关联新商户号，输入 1113511352
  4. 确认关联

  两步都完成后，wx.requestPayment 就不会再返回 access denied 了。

  ▎ 注意：如果之前支付是正常的（这次部署后才出问题），检查一下是否 PAYMENT_SANDBOX_MODE 
  ▎ 或后端微信配置被覆盖了。目前服务器配置是生产模式，mock 关闭，APIv3 密钥和证书路径都在。
