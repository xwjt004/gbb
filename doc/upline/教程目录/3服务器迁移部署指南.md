## 1. 前置准备

### 1.1 服务器信息

| 项目 | 值 |
|------|-----|
| 新服务器 IP | `81.68.236.12` |
| 新域名 | `guaibaobao.cn` |
| SSH 用户 | `ubuntu` |
| SSH 密钥 | `~/.ssh/deploy_key` |
| 操作系统 | Ubuntu (腾讯云) |
| 项目根目录 | `/opt/gbb/` |
| 后端构建目录 | `/app/backend-build/` |
| PostgreSQL 数据目录 | `/app/postgres/data/` |

### 1.2 本地 SSH 配置

编辑本地 `~/.ssh/config`，添加服务器快捷登录（可复制以下内容）：

```
Host gbb-server
  HostName 81.68.236.12
  User ubuntu
  IdentityFile ~/.ssh/deploy_key
  StrictHostKeyChecking no
```

**命令说明：**
- `Host`：自定义主机别名，之后可以用 `ssh gbb-server` 代替完整命令
- `IdentityFile`：指定私钥文件路径，SSH 用这个密钥来验证身份
- `StrictHostKeyChecking no`：跳过首次连接的主机指纹确认（安全要求不高的内网/测试环境可用）

配置好后，登录只需：
```bash
ssh gbb-server      # 登录服务器
```

### 1.3 确保 SSH 密钥权限正确

```bash
chmod 600 ~/.ssh/deploy_key
```

**为什么：** SSH 要求私钥文件只有文件所有者可读写，权限过宽会报 `Permissions 0644 for 'deploy_key' are too open`。

## 2. 服务器初始化

### 2.1 SSH 登录

```bash
ssh -i ~/.ssh/deploy_key -o StrictHostKeyChecking=no ubuntu@81.68.236.12
```

命令参数说明：
- `-i ~/.ssh/deploy_key`：指定使用的身份密钥文件
- `-o StrictHostKeyChecking=no`：首次连接不提示确认主机指纹
- `ubuntu@81.68.236.12`：以 ubuntu 用户登录到目标服务器

### 2.2 安装 Docker

> **注意：** 腾讯云服务器从 Docker 官方源下载会超时（国内网络限制），需要使用腾讯云镜像源。

```bash
# 安装 Docker（使用腾讯云镜像源）
curl -fsSL https://mirrors.tencent.com/docker-ce/linux/ubuntu/gpg | sudo apt-key add -
sudo add-apt-repository "deb [arch=amd64] https://mirrors.tencent.com/docker-ce/linux/ubuntu $(lsb_release -cs) stable"
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin
```

**每行作用：**
- `curl ... | sudo apt-key add -`：下载 Docker 官方的 GPG 签名密钥，用于验证软件包完整性
- `add-apt-repository`：添加 Docker 软件源到 apt 源列表
- `lsb_release -cs`：自动获取当前 Ubuntu 版本代号（如 focal, jammy）
- `apt-get install`：安装 Docker 引擎和相关组件
- `docker-compose-plugin`：Docker Compose 的插件版本，命令为 `docker compose`（带空格）

### 2.3 配置 Docker 镜像加速

国内拉取 Docker 镜像很慢（官方 registry 被限速/屏蔽），需要配置镜像加速器：

```bash
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json << 'EOF'
{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com",
    "https://docker.m.daocloud.io"
  ]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker
```

**什么是镜像加速器：** Docker Hub 的国内缓存代理，拉取镜像时从加速器下载而不是直接连 Docker Hub，速度快很多。腾讯云的加速器只对腾讯云服务器有效。

### 2.4 验证 Docker 安装

```bash
docker --version
docker compose version
docker run hello-world
```

## 3. 运行数据库和缓存

### 3.1 创建 Docker 网络

容器之间需要通过 Docker 内部网络通信，不能依赖 IP 地址（容器重启后 IP 会变），要用容器名称：

```bash
docker network create app_gbb-network
```

**为什么需要自定义网络：** Docker 默认网络不支持容器名解析。自定义网络中，容器可以通过 `--name` 设置的名称互相访问（如 `gbb-backend` 可以通过 `redis` 这个主机名连接到 Redis 容器）。

### 3.2 启动 PostgreSQL

```bash
docker run -d \
  --name gbb-postgres \
  --network app_gbb-network \
  -p 5432:5432 \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=baby_photo_db \
  -v /app/postgres/data:/var/lib/postgresql/data \
  --restart unless-stopped \
  postgres:17-alpine
```

**参数详解：**
- `-d`：后台运行（detach）
- `--name`：容器名称，其他容器可通过这个名字访问它
- `--network`：加入自定义网络
- `-p 5432:5432`：宿主机端口:容器端口映射。宿主机 5432 → 容器内 5432
- `-e`：设置环境变量
  - `POSTGRES_USER`：数据库超级用户
  - `POSTGRES_PASSWORD`：数据库密码
  - `POSTGRES_DB`：初始创建的数据库名（仅在首次启动时创建）
- `-v 宿主机目录:容器目录`：数据持久化。容器删除后数据不丢失
- `--restart unless-stopped`：容器退出时自动重启，除非手动停止
- `postgres:17-alpine`：使用 PostgreSQL 17 的 Alpine Linux 精简版镜像

### 3.3 启动 Redis

```bash
docker run -d \
  --name gbb-redis \
  --network app_gbb-network \
  -p 6379:6379 \
  --restart unless-stopped \
  redis:7-alpine
```

Redis 默认不需要密码，如果后端 `.env` 中配置了 `REDIS_PASSWORD`，需要加 `-e REDIS_PASSWORD=xxx`。

## 4. 备份/迁移数据库

### 4.1 从旧服务器导出并导入（核心操作）

```bash
# 在本地电脑执行（不要 SSH 到服务器）
# 这条命令将旧服务器的数据库直接管道传输到新服务器，不经过本地磁盘
ssh -i ~/.ssh/deploy_key gbb-server \
  "docker exec gbb-postgres pg_dump -U postgres baby_photo_db --no-owner" \
  | ssh -i ~/.ssh/deploy_key ubuntu@81.68.236.12 \
  "docker exec -i gbb-postgres psql -U postgres -d baby_photo_db"
```

**这条命令的工作方式（管道传输）：**
1. `ssh ... gbb-server "docker exec ... pg_dump"`：SSH 登录旧服务器，在 PostgreSQL 容器内执行 `pg_dump` 导出数据库
2. `|`（管道符）：将 `pg_dump` 的输出（SQL 语句）直接传给下一个命令
3. `ssh ... ubuntu@81.68.236.12 "docker exec ... psql"`：SSH 登录新服务器，在 PostgreSQL 容器内执行 `psql` 导入 SQL

**为什么用管道而不是先导出成文件：** 避免中间文件占用磁盘空间，也更快。
**为什么加 `--no-owner`：** 新旧服务器的数据库用户可能不同，不加这个参数会导致导入时因找不到原用户而报错。

### 4.2 验证数据完整性

```bash
# 查看有哪些表
docker exec gbb-postgres psql -U postgres -d baby_photo_db -c '\dt'

# 查看用户表数据行数
docker exec gbb-postgres psql -U postgres -d baby_photo_db -c 'SELECT count(*) FROM users;'

# 查看订单表数据行数
docker exec gbb-postgres psql -U postgres -d baby_photo_db -c 'SELECT count(*) FROM orders;'
```

**`psql` 常用命令：**
- `\dt`：列出所有表（list tables）
- `\du`：列出所有用户/角色
- `\l`：列出所有数据库
- `\d 表名`：查看表结构
- `\dn`：列出所有 schema
- `SELECT count(*) FROM 表名`：查看数据行数

### 4.3 修复用户权限

从 `pg_dump` 导入的数据，所有者是旧服务器的用户（如 `postgres`），新服务器需要重新授权：

```bash
# 创建新用户（如果不存在）
docker exec gbb-postgres psql -U postgres -c "CREATE ROLE baby_photo WITH LOGIN PASSWORD 'baby_photo_2024' CREATEDB;"

# 授权数据库
docker exec gbb-postgres psql -U postgres -c "GRANT ALL PRIVILEGES ON DATABASE baby_photo_db TO baby_photo;"

# 授权所有表
docker exec gbb-postgres psql -U postgres -d baby_photo_db -c "GRANT ALL ON ALL TABLES IN SCHEMA public TO baby_photo;"
docker exec gbb-postgres psql -U postgres -d baby_photo_db -c "GRANT ALL ON ALL SEQUENCES IN SCHEMA public TO baby_photo;"
docker exec gbb-postgres psql -U postgres -d baby_photo_db -c "GRANT ALL ON SCHEMA public TO baby_photo;"
docker exec gbb-postgres psql -U postgres -d baby_photo_db -c "ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON TABLES TO baby_photo;"
docker exec gbb-postgres psql -U postgres -d baby_photo_db -c "ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON SEQUENCES TO baby_photo;"

# 更改表所有者为新用户（关键！否则 Prisma 无法执行迁移）
# 先生成 ALTER TABLE 命令，再执行
docker exec gbb-postgres psql -U postgres -d baby_photo_db -t \
  -c "SELECT 'ALTER TABLE '||tablename||' OWNER TO baby_photo;' FROM pg_tables WHERE schemaname='public';" \
  | xargs -I {} docker exec gbb-postgres psql -U postgres -d baby_photo_db -c "{}"
```

**为什么需要改所有权：** Prisma 在 `migrate deploy` 时需要读写 `_prisma_migrations` 表。如果表所有者还是 `postgres`，普通用户 `baby_photo` 无法修改这个表，迁移会失败。

## 5. 部署后端

### 5.1 准备 Dockerfile

创建 `/app/backend-build/Dockerfile`：

```dockerfile
FROM node:20-alpine
WORKDIR /app

# 安装系统依赖
# wget: 健康检查用  openssl: Prisma 需要  redis: 健康检查需要 redis-cli
RUN apk add --no-cache wget openssl redis

# 安装 Node 依赖
COPY package.json package-lock.json ./
RUN npm ci --ignore-scripts && npm cache clean --force

# 生成 Prisma 客户端（根据 schema 生成类型安全的数据库客户端代码）
COPY prisma ./prisma
RUN npx prisma generate

# 复制编译后的代码
COPY dist ./dist
COPY docker-entrypoint.sh ./
RUN chmod +x docker-entrypoint.sh

EXPOSE 3000
ENTRYPOINT ["/bin/sh", "docker-entrypoint.sh"]
```

**`apk add` 各包的作用：**
- `wget`：Docker 健康检查时用来请求后端接口
- `openssl`：Prisma 连接 PostgreSQL 时需要
- `redis`：提供 `redis-cli` 命令，健康检查用

### 5.2 准备 docker-entrypoint.sh

```bash
#!/bin/sh
set -e

echo "=== Running database migrations ==="
npx prisma migrate deploy 2>&1

echo "=== Starting application ==="
exec node dist/main.js
```

**`set -e`：** 脚本中任何命令失败就立即退出，避免在数据库迁移失败后继续启动应用。

### 5.3 构建 Docker 镜像

```bash
# 在后端源码目录下执行（包含 Dockerfile 的目录）
cd /app/backend-build
docker build -t baby-photo-backend:latest .
```

**`-t baby-photo-backend:latest`：** 给镜像打标签（tag），`:` 前面是仓库名，后面是版本标签

### 5.4 启动后端容器

```bash
docker run -d \
  --name gbb-backend \
  --network app_gbb-network \
  -p 3000:3000 \
  -e DATABASE_URL='postgresql://baby_photo:baby_photo_2024@postgres:5432/baby_photo_db' \
  -e REDIS_HOST=redis \
  -e REDIS_PORT=6379 \
  -e REDIS_PASSWORD='' \
  -e JWT_SECRET='gbb-server-jwt-secret-2024-random-xyz-abc-9876543210' \
  -e CORS_ORIGIN='http://localhost:3001,https://guaibaobao.cn' \
  -e NODE_ENV=production \
  --restart unless-stopped \
  baby-photo-backend:latest
```

**环境变量说明：**

| 变量 | 作用 | 从哪里获取 |
|------|------|-----------|
| `DATABASE_URL` | 数据库连接字符串 `postgresql://用户名:密码@主机:端口/数据库名` | 自己设定的 PostgreSQL 用户密码 |
| `REDIS_HOST` | Redis 主机名（Docker 网络内直接用容器名 `redis`） | 启动 Redis 容器时指定的 `--name` |
| `JWT_SECRET` | 登录令牌签名密钥，必须是一个足够长（32位以上）的随机字符串 | 自己生成 |
| `CORS_ORIGIN` | 允许跨域访问的前端地址列表，逗号分隔 | 实际使用的前端域名 |
| `NODE_ENV` | 运行环境，`production` 会关闭 Swagger 文档等调试功能 | - |

**DATABASE_URL 格式拆解：**
```
postgresql://baby_photo:baby_photo_2024@postgres:5432/baby_photo_db
             └─┬──┘    └─────┬─────┘  └───┬──┘ └─┬┘ └─────┬──────┘
             用户名          密码       主机  端口   数据库名
```

### 5.5 验证后端

```bash
# 检查容器状态
docker ps --format 'table {{.Names}}\t{{.Status}}'

# 查看启动日志
docker logs gbb-backend

# 测试健康检查接口（应返回 {"status":"ok"}）
curl -s http://localhost:3000/api/v1/health | python3 -m json.tool

# 测试业务接口
curl -s http://localhost:3000/api/v1/users?page=1 | python3 -m json.tool
```

**健康检查响应示例：**
```json
{
    "status": "ok",
    "services": {
        "database": "connected",
        "redis": "connected",
        "disk": { "total": "49.1GB", "used": "8.1GB", "free": "38.9GB" }
    }
}
```

如果 Redis 显示 "disconnected"，检查：
- Redis 容器是否在运行：`docker ps | grep redis`
- 后端能否连通 Redis：`docker exec gbb-redis redis-cli ping`（应返回 PONG）
- 如果 `redis-cli` 命令找不到，说明 Docker 镜像中没安装 redis 包，需要在 Dockerfile 的 `apk add` 中加入 `redis`

## 7. 配置 Nginx 反向代理

### 7.1 创建 Nginx 配置

`/opt/gbb/nginx/nginx.conf`：

```nginx
# HTTP → HTTPS 重定向
server {
    listen 80;
    server_name guaibaobao.cn www.guaibaobao.cn;
    return 301 https://$server_name$request_uri;
}

# HTTPS 服务器
server {
    listen 443 ssl http2;
    server_name guaibaobao.cn www.guaibaobao.cn;

    ssl_certificate     /etc/letsencrypt/live/guaibaobao.cn/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/guaibaobao.cn/privkey.pem;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;

    root /usr/share/nginx/html;
    index index.html;

    # 管理后台 (SPA)
    location /admin/ {
        root /usr/share/nginx/html;
        try_files $uri $uri/ /admin/index.html;
    }

    # Gzip 压缩：传输前压缩文本类文件，减少带宽
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml
               application/xml text/javascript image/svg+xml;
    gzip_min_length 1000;

    # 静态资源缓存：/assets/ 下的文件带哈希值，可以永久缓存
    location /assets/ {
        expires 1y;
        add_header Cache-Control "public";
    }

    # SPA 单页应用路由：所有非文件路径都返回 index.html
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理：将 /api/v1/ 的请求转发到后端
    location /api/v1/ {
        proxy_pass http://gbb-backend:3000/api/v1/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 120s;
        proxy_send_timeout 120s;
    }

    # 上传文件访问代理：图片等静态文件由后端提供
    location /uploads/ {
        proxy_pass http://gbb-backend:3000/uploads/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        expires 7d;
        add_header Cache-Control "public";
    }
}
```

**关键配置说明：**

| 指令 | 作用 |
|------|------|
| `proxy_pass` | 将请求转发到后端地址，`gbb-backend:3000` 是在 Docker 网络内通过容器名访问 |
| `proxy_set_header Host $host` | 把原始域名传给后端，后端根据 Host 判断请求来源 |
| `proxy_set_header X-Real-IP` | 把客户端真实 IP 传给后端（否则后端看到的是 Nginx 的 IP）|
| `proxy_set_header X-Forwarded-For` | 如果经过多层代理，记录完整的 IP 链路 |
| `try_files $uri $uri/ /index.html` | 先找真实文件，找不到就返回 index.html（前端路由需要）|

### 7.2 启动 Nginx 容器

```bash
docker run -d \
  --name gbb-nginx \
  --network app_gbb-network \
  -p 80:80 \
  -p 443:443 \
  -v /opt/gbb/www:/usr/share/nginx/html:ro \
  -v /opt/gbb/nginx/nginx.conf:/etc/nginx/conf.d/default.conf:ro \
  -v /etc/letsencrypt:/etc/letsencrypt:ro \
  --restart unless-stopped \
  nginx:alpine
```

**卷挂载（`-v`）说明：**
- `/opt/gbb/www:/usr/share/nginx/html:ro`：前端静态文件目录（`www/admin/` 为管理后台），`:ro` 表示只读
- `/opt/gbb/nginx/nginx.conf:/etc/nginx/conf.d/default.conf:ro`：Nginx 配置映射
- `/etc/letsencrypt:/etc/letsencrypt:ro`：SSL 证书映射（Let's Encrypt 证书路径）

## 8. SSL 证书

### 8.1 申请证书（已生效）

当前服务器已使用 Let's Encrypt 申请并配置了 SSL 证书，证书信息如下：

## 12. 故障排查

### 12.1 后端容器不断重启

```bash
# 查看最新日志
docker logs gbb-backend --tail 50
```

**常见原因：**
- **缺少环境变量**：日志显示 "缺少以下必需环境变量：JWT_SECRET"。→ 在 docker run 时加 `-e JWT_SECRET=xxx`
- **数据库连接失败**：日志显示 "Authentication failed"。→ 检查 DATABASE_URL 的用户名密码是否正确，数据库是否已创建
- **Prisma 迁移失败**：日志显示 "permission denied for table _prisma_migrations"。→ 需要执行 4.3 节的授权命令

### 12.2 前端白屏 / 路由不工作

- **检查 Nginx 日志**：`docker logs gbb-nginx --tail 50`
- **检查 `try_files` 配置**：确保 `location /` 中有 `try_files $uri $uri/ /index.html;`
- **检查 API 代理**：如果前端能加载但接口 502，可能是 Nginx 连不上后端，检查 `proxy_pass` 地址是否正确

### 12.3 Nginx 启动失败

```bash
docker logs gbb-nginx
```

**常见原因：**
- `"invalid parameter"`：Nginx 版本不支持某些配置参数（如旧版 nginx 不支持 `immutable` 参数）。→ 去掉不支持的参数
- `"proxy_set_header" directive invalid number of arguments`：变量被 shell 展开成了空值。→ 检查 nginx.conf 中 `$host`、`$remote_addr` 等变量是否被正确保留（使用单引号 heredoc 写入文件）

### 12.4 SSH 管道传输数据库中断

```bash
# 如果管道传输中断，先确认目标数据库是空的，再重新执行
# 重新创建干净的数据库
docker exec gbb-postgres psql -U postgres -c "DROP DATABASE IF EXISTS baby_photo_db;"
docker exec gbb-postgres psql -U postgres -c "CREATE DATABASE baby_photo_db OWNER baby_photo;"
# 然后重新执行管道传输命令
```

### 12.5 磁盘空间不足

```bash
# 查看磁盘使用
df -h

# 清理不用的 Docker 资源
docker system prune -f        # 清理停止的容器、不用的网络和 dangling 镜像
docker system prune -a -f     # 清理所有不用的镜像（包括未被容器使用的）

# 查看各目录大小
sudo du -sh /opt/gbb/*
sudo du -sh /app/*
```
