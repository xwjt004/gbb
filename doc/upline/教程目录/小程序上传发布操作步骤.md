# 微信小程序上传发布操作步骤

---

## 第一步：在开发者工具中上传代码

1. 打开 **微信开发者工具**
2. 打开项目路径：`D:\vscode\GBBV2\client\psyd`
3. 点击顶部工具栏的 **「上传」** 按钮（云朵图标 ↑）

   ![上传按钮位置](说明：顶部工具栏右侧，与"预览""清楚缓存"同级)

4. 弹窗中填写：
   - **版本号**：例如 `2.1.0`（建议遵循语义化版本：主版本.次版本.修订号）
   - **项目备注**：例如 `feat: 新增会员中心、个人编辑、手机号登录`
5. 点击 **「上传」**，等待进度条完成

---

## 第二步：到微信公众平台管理版本

1. 浏览器打开：**[https://mp.weixin.qq.com](https://mp.weixin.qq.com)**
2. 扫码登录你的小程序管理后台
3. 左侧菜单选择 **「管理」→「版本管理」**

此时你应该能看到刚才上传的版本出现在 **「开发版本」** 区域。

---

## 第三步：设置为体验版（可选，用于测试）

在「开发版本」区域，找到刚上传的版本，点击右侧 **「设为体验版」**。

- 设置后，**体验版二维码** 会出现在页面中
- 将二维码发给已授权的微信号（在「成员管理」中添加），他们扫码即可预览未发布的版本
- 体验版适合内部测试验收

---

## 第四步：提交审核（发布到线上）

测试无误后，在「开发版本」区域点击 **「提交审核」**：

1. 填写 **审核说明**（让微信审核人员知道这次改了啥）
   ```
   新增功能：会员中心、个人资料编辑、手机号登录
   页面优化：登录页、优惠券页、套系列表页美化
   ```
2. 勾选 **「本次更新涉及哪些功能」**（按实际勾选）
3. 点击 **「提交审核」**

---

## 第五步：审核通过后发布

1. 微信审核通常需要 **30分钟 ~ 24小时**
2. 审核通过后，微信会发送 **服务通知** 到小程序管理员微信
3. 回到 **版本管理** 页面，在「审核通过版本」区域点击 **「发布」**
4. 确认发布，等待数分钟即可全网生效

---

## 版本流程图解

```
开发者工具上传
     ↓
【开发版本】  ← 可设为体验版供测试
     ↓
  提交审核
     ↓
【审核中】    ← 等待微信审核（约30min~24h）
     ↓
  审核通过
     ↓
【审核通过版本】← 此时可点击"发布"
     ↓
【线上版本】  ← 所有用户可见
```

---

## 注意事项

| 注意点 | 说明 |
|--------|------|
| 版本号递增 | 每次上传版本号要比上次大，建议：`2.1.0` → `2.1.1` → `2.2.0` |
| 备注写清楚 | 方便后续回溯该版本包含哪些改动 |
| 审核被拒 | 查看拒绝原因，修改后重新提交 |
| 紧急回退 | 版本管理中有「撤回」功能，可回退到上一个线上版本 |
| 正式发布 | 建议避开周五晚上和周末提交审核，审核周期会更长 |
