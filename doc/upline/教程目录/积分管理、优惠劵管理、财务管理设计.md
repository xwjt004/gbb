## 一、微信小程序积分功能

### 1.1 积分兑换比例

| 项目 | 值 |
|------|-----|
| 基础比例 | 100 积分 = ¥0.01（1分钱） |
| 折算 | 10000 积分 = ¥1（1元钱） |
| 单个积分价值 | 1 积分 = ¥0.0001 |
| 获取方式 | 订单支付（1元=1基础分 + 10奖励分）、每日签到（+2）、后台手动调整 |
| 使用方式 | 结算时抵扣订单金额 |

### 1.2 数据库设计

#### PointsTransaction 模型（新增）

```prisma
model PointsTransaction {
  id          String   @id @default(cuid())
  wxUserId    String   @map("wx_user_id")
  type        String   // EARN / SPEND / EXPIRE / ADMIN
  amount      Int      // 正=获得, 负=消耗
  balance     Int      // 交易后余额
  source      String?  // ORDER / CHECKIN / REDEEM / ADMIN
  sourceId    String?  // 关联单号(order_no)
  description String?  @db.Text
  createdAt   DateTime @default(now()) @map("created_at")
  wxUser      WxUser   @relation(fields: [wxUserId], references: [id])
  @@index([wxUserId, createdAt])
  @@map("points_transactions")
}
```

#### Order 模型新增字段

```prisma
  pointsUsed          Int      @default(0)   // 使用的积分数量
  pointsDeductAmount  Decimal  @default(0)   // 积分抵扣金额
```

### 1.3 后端 API 设计

#### 小程序端 API（WxJwtAuthGuard 保护）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/wx-points/my?page=&limit=` | 返回积分余额 + 分页记录 |
| GET | `/wx-points/rules` | 积分规则（兑换比例、获取方式） |
| POST | `/wx-points/checkin` | 每日签到 +2 积分 |

#### 管理后台 API（JwtAuthGuard 保护）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/admin/points/transactions` | 积分记录列表（支持多条件筛选） |

### 1.4 核心业务流程

#### 订单支付 → 积分发放
```
支付成功回调(FULLY_PAID)
  → awardOrderPoints()
    → 计算积分 = Math.floor(totalAmount) × 1 + 10
    → 事务: 创建 PointsTransaction(EARN) + 更新 WxUser.growthPoints
```

#### 结算时积分抵扣
```
用户点击"使用积分"
  → 加载当前积分余额
  → 计算最大可抵扣 = min(余额, 订单金额 / 0.0001)
  → 用户确认使用数量
  → 提交订单时传入 pointsUsed
  → 后端:
    1. 验证余额充足
    2. pointsDeductAmount = Math.floor(pointsUsed × 0.0001 × 100) / 100
    3. finalAmount = totalAmount - discountAmount - pointsDeductAmount
    4. 事务: 创建 PointsTransaction(SPEND) + 更新 WxUser.growthPoints
```

## 三、财务管理模块

### 3.1 设计目标

在现有仅统计营收（revenue）的基础上，加入成本追踪和利润计算，让管理者能清楚知道：

| 指标 | 说明 |
|------|------|
| 营业收入 | 实收金额（paidAmount），即客户实际支付的现金 |
| 营业成本 | totalCostAmount，即商品/服务的成本总和 |
| 营业利润 | profitAmount = paidAmount - totalCostAmount |
| 利润率 | profitRate = profitAmount / paidAmount × 100% |

### 3.2 数据库设计

#### Product 模型（已有字段）
```prisma
  costPrice   Decimal  @map("cost_price")  // 商品成本价（已有，需确保数据完整）
```

#### OrderItem 模型新增字段
```prisma
  costPrice   Decimal  @default(0)  // 创建订单时的成本价快照
```

#### Order 模型新增字段
```prisma
  totalCostAmount  Decimal  @default(0)  // 总成本（∑ OrderItem.costPrice × quantity）
  profitAmount     Decimal  @default(0)  // 利润 = paidAmount - totalCostAmount
```

### 3.3 后端 API 设计

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/admin/finance/profit-overview` | 利润概览（营收/成本/利润/趋势图） |
| GET | `/admin/finance/profit-detail` | 分页利润明细（每单利润） |

### 3.4 核心业务流程

#### 订单创建时
- 遍历订单项，对 PRODUCT 类型从 `Product.costPrice` 读取成本
- 写入 `OrderItem.costPrice`（快照，防止后续改价影响历史）
- 计算 `totalCostAmount = ∑ costPrice × quantity`

#### 支付完成时
- `profitAmount = paidAmount - totalCostAmount`
- 更新 Order 的 `totalCostAmount` 和 `profitAmount`

#### 报表查询
- 按日期聚合：`SELECT date, SUM(paidAmount), SUM(totalCostAmount), SUM(profitAmount)`
- 趋势图：每日/每周/每月 营收 vs 成本 vs 利润

### 3.5 管理后台页面

#### 利润概览（`/finance/profit`）
- 统计卡片：本期营收、总成本、利润、利润率
- 趋势折线图（ECharts）：每日营收/成本/利润
- 日期范围筛选

#### 成本明细（`/finance/cost-detail`）
- 表格：订单号、商品、售价、成本、利润、利润率、支付时间
- 支持筛选：日期范围、支付状态

### 3.6 历史数据补丁

上线前对已有订单执行一次性脚本，补充 `OrderItem.costPrice` 和 `Order.totalCostAmount`/`profitAmount`。
