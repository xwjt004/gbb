## 一、前置概念

要让小程序支持微信支付，需要三个东西配合工作：

```
小程序（AppID）  ←→  商户号（商户号）  ←→  后端服务器
    ↑                     ↑
 微信那边              你在微信支付的"账户"
 的身份ID              用来收钱、退款、对账
```

## 三、你的项目是否需要开通微信支付？

**答案是：是的，如果你要在线收款的话。**

你们的业务模式：
- 客户在小程序选套系 → 预约 → **在线付定金/全款**
- 如果不接支付，客户只能线下到店付款，体验不完整

**如果你暂时不想接：**
也可以先用「到店付」模式——客户在小程序预约，到店后再扫码付款或现金。但这样小程序的价值就少了一半。

## 四、开通微信支付的完整流程

### 第一步：申请商户号

打开 **[pay.weixin.qq.com](https://pay.weixin.qq.com)** → 点击「注册」→ 选择「个体工商户」或「企业」。

**需要准备的材料：**

| 主体类型 | 需要的材料 |
|---------|-----------|
| **个体工商户** | 营业执照、经营者身份证、对公账户或法人银行卡 |
| **企业** | 营业执照、法人身份证、对公账户 |

**审核时间：** 1~3 个工作日。

### 第二步：关联商户号到小程序

拿到商户号后，回到 **mp.weixin.qq.com → 微信支付** 页面：

1. 点击「申请关联商户号」
2. 输入你的商户号
3. 按提示扫码确认
4. 等待审核（一般即时生效）

### 第三步：配置支付密钥

登录 **[pay.weixin.qq.com](https://pay.weixin.qq.com)** → 「账户中心」→「API 安全」：

| 需要配置项 | 用途 |
|-----------|------|
| **API 密钥（API Key）** | 后端签名支付请求，必须配 |
| **APIv3 密钥** | 新版支付签名，建议也配 |
| **证书下载** | 用于退款、转账等 API |

配好后把这些信息更新到服务器 `.env` 文件中：

```
WX_MCH_ID=你的商户号
WX_API_KEY=API密钥
WX_API_V3_KEY=APIv3密钥
WX_CERT_PATH=/app/certs/apiclient_cert.p12
```

### 第四步：配置支付回调域名

在 **mp.weixin.qq.com → 开发管理 → 开发设置 → 服务器域名** 确认一下：

目前已经是：`https://guaibaobao.cn`

支付回调一般也用这个域名，已配好。

### 第五步：配置支付授权目录

登录 **[pay.weixin.qq.com](https://pay.weixin.qq.com)** → 「产品中心」→「开发配置」：

- **JSAPI 支付授权目录**：`https://guaibaobao.cn/`
- **Native 支付回调链接**：`https://guaibaobao.cn/api/v1/wx-pay/notify`

## 五、后端需要配合的工作

支付功能需要后端支持，你们项目里已经有相应的模块（`wx-pay`），只需要商户号配置好之后：

1. 在服务器 `.env` 填入商户号和相关密钥
2. 重启后端容器即可生效

## 六、下一步建议

| 优先级 | 事项 | 预计耗时 |
|-------|------|---------|
| ⭐ 高 | **申请微信支付商户号**（营业执照等材料准备好） | 1~3 天 |
| 中 | 商户号下来后，关联到小程序 | 10 分钟 |
| 中 | 配置 API 密钥和证书 | 30 分钟 |
| 低 | 后端 `.env` 更新商户配置 | 10 分钟 |

## 七、常见问题

**Q：个体户可以申请吗？**
A：可以。用个体工商户营业执照即可，不需要注册公司。

**Q：没有营业执照怎么办？**
A：那就开不了微信支付。可以先去当地市场监督管理局办个个体工商户执照，流程很简单，一般在 APP 上就能办。

**Q：申请商户号要钱吗？**
A：注册免费。每笔交易微信收手续费（通常 0.6%）。

**Q：可以先上线小程序，以后再接入支付吗？**
A：可以。小程序可以先展示商品和预约，支付按钮先隐藏或提示「到店付款」，等商户号下来后再开启。
