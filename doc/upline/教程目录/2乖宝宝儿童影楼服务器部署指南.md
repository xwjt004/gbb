## 2. 如何连接服务器

### 2.1 使用 SSH 密钥连接（推荐方式）

在本地电脑（Windows / Mac）的终端中执行：

```bash
# 使用已配置的密钥文件连接服务器
ssh -i ~/.ssh/deploy_key root@107.175.69.17
```

> **参数说明**  
> - `-i ~/.ssh/deploy_key`：指定用于身份验证的私钥文件（服务器认识这把钥匙）  
> - `root`：以超级管理员身份登录  
> - `107.175.69.17`：服务器 IP 地址

首次连接会提示确认主机指纹，输入 `yes` 回车即可。后面再连接就不会再提示了。

### 2.2 使用 SSH 别名连接（推荐，更快捷）

编辑本地 `~/.ssh/config` 文件，添加以下内容：

```ini
Host gbb-server
  HostName 107.175.69.17
  User root
  IdentityFile ~/.ssh/deploy_key
  StrictHostKeyChecking no
```

添加后，只需要输入以下命令即可连接服务器：

```bash
ssh gbb-server
```

> **解释**：`~/.ssh/config` 是 SSH 客户端的配置文件，可以给服务器起别名、预设登录参数。`Host gbb-server` 后面的配置就是这个别名的连接参数。配置好后，`ssh gbb-server` 就相当于 `ssh -i ~/.ssh/deploy_key root@107.175.69.17` 的简写。

### 2.3 连接后的第一个检查

登录服务器后，先确认服务是否正常运行：

```bash
# 查看所有 Docker 容器状态
docker ps

# 正常应该看到类似这样（全部 Up 且 healthy）：
# NAMES                STATUS
# gbb-backend          Up 2 hours (healthy)
# gbb-postgres         Up 2 hours (healthy)
# gbb-redis            Up 2 hours (healthy)
# gbb-nginx            Up 2 hours
# gbb-admin-frontend   Up 2 hours
```

如果某个容器状态异常（如 `unhealthy` 或 `Restarting`），参考[常见问题排查](#10-常见问题排查)处理。

1. **永远不要**将 `.env` 文件提交到 Git 仓库（已在 `.gitignore` 中排除）
2. **定期备份**：将 `.env` 内容备份到安全的位置（如密码管理器）
3. **修改后必须重启后端**才能生效：参照 5.2 节的"修改配置后重启"流程

## 6. SSL/HTTPS 证书

### 6.1 当前使用的方案

我们使用 **Cloudflare Full 模式**，即：
- 用户浏览器 ↔ Cloudflare：标准 HTTPS（由 Cloudflare 自动管理证书）
- Cloudflare ↔ 服务器（nginx）：使用**自签名证书**

自签名证书位于 `/opt/baby-photo/ssl/` 目录下：

```bash
ls -la /opt/baby-photo/ssl/
# cert.pem  — 证书文件（公钥）
# key.pem   — 私钥文件（保密）
```

### 6.2 证书到期后如何续期

自签名证书有效期 **365 天**。到期后重新生成：

```bash
cd /opt/baby-photo

# 备份旧证书
cp ssl/cert.pem ssl/cert.pem.bak
cp ssl/key.pem ssl/key.pem.bak

# 重新生成证书
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout ssl/key.pem \
  -out ssl/cert.pem \
  -subj "/CN=guaibaobao.net" \
  -addext "subjectAltName=DNS:guaibaobao.net,DNS:www.guaibaobao.net"

# 重启 nginx 加载新证书
docker exec gbb-nginx nginx -s reload
```

> **命令解释**  
> `openssl` 是 OpenSSL 工具，用于生成和管理 SSL 证书。  
> `req -x509`：创建自签名证书（非 CA 签发）。  
> `-days 365`：有效期 1 年。  
> `-newkey rsa:2048`：同时生成新私钥，2048 位 RSA 加密。  
> `-nodes`：不对私钥加密（no DES），nginx 启动时无需输入密码。  
> `-subj`：证书所属主体，`/CN=` 后跟域名。  
> `-addext`：添加 SAN（Subject Alternative Name），浏览器要求此字段包含实际域名。

### 6.3 切换到 Let's Encrypt（可选）

如果需要浏览器直接信任的证书（不经过 Cloudflare），可以使用 Let's Encrypt：

```bash
# 安装 certbot
apt install -y certbot

# 临时在 80 端口申请证书（需要先停掉 nginx）
docker stop gbb-nginx
certbot certonly --standalone -d guaibaobao.net -d www.guaibaobao.net

# 复制证书到项目目录
cp /etc/letsencrypt/live/guaibaobao.net/fullchain.pem /opt/baby-photo/ssl/cert.pem
cp /etc/letsencrypt/live/guaibaobao.net/privkey.pem /opt/baby-photo/ssl/key.pem

# 重启 nginx
docker compose up -d nginx

# 设置自动续期（证书有效期 90 天，每天凌晨 3 点检查）
crontab -e
# 添加以下行：
0 3 * * * certbot renew --quiet && docker exec gbb-nginx nginx -s reload
```

### 7.1 数据库基本信息

| 内部端口 | 5432 |
| 外部端口 | 127.0.0.1:5432（仅本地可访问） |

### 7.2 进入数据库命令行

```bash
# 方式一：在服务器上直接进入
docker exec -it gbb-postgres psql -U postgres -d baby_photo_db

# 方式二：从本地机器连接（需要 SSH 隧道）
# 先用 SSH 隧道把服务器数据库端口转发到本地
ssh -i ~/.ssh/deploy_key -L 5433:127.0.0.1:5432 root@107.175.69.17 -N &
# 然后本地用数据库工具连接 localhost:5433
```

> **什么是 SSH 隧道？**  
> 数据库只监听 127.0.0.1（本地），外部无法直接连接。SSH 隧道相当于在本地和服务器之间建立一条加密通道，把本地的 5433 端口转发到服务器的 5432 端口，这样本地工具（如 Navicat、DataGrip）就能连接了。`-N` 表示不执行远程命令（只做转发），`&` 让它在后台运行。

### 7.3 常用 SQL 操作

```sql
-- 查看所有表
\dt

-- 查看表结构
\d 表名

-- 查看迁移记录
SELECT * FROM _prisma_migrations;

-- 查询用户
SELECT id, nickname, phone, role FROM users;

-- 查询订单
SELECT id, order_no, order_status, total_amount, created_at FROM orders 
ORDER BY created_at DESC LIMIT 10;
```

### 7.4 数据库备份与恢复

```bash
# 创建备份目录
mkdir -p /opt/baby-photo/backups

# ===== 备份数据库 =====
docker exec gbb-postgres pg_dump -U postgres baby_photo_db > \
  /opt/baby-photo/backups/baby_photo_$(date +%Y%m%d_%H%M%S).sql

# 备份后文件在 /opt/baby-photo/backups/ 目录下，文件名带时间戳

# ===== 恢复数据库（谨慎操作！会覆盖当前数据）=====
docker exec -i gbb-postgres psql -U postgres -d baby_photo_db < \
  /opt/baby-photo/backups/baby_photo_20260510_120000.sql
```

> **`pg_dump` 说明**：PostgreSQL 自带的备份工具，导出为纯 SQL 文件，包含完整的 CREATE TABLE 和 INSERT 语句，可在任何 PostgreSQL 版本上恢复。建议定期备份（可设置 cron 定时任务）。

### 7.5 Prisma 迁移

数据库结构变更通过 Prisma 迁移管理：

```bash
# 查看迁移状态（进入后端容器）
docker exec -it gbb-backend /bin/sh
cd /app
npx prisma migrate status

# 查看已应用的迁移列表
```

### 9.3 本地代码修改后如何同步到服务器

## 10. 常见问题排查

### 10.1 网站打不开（502 Bad Gateway）

**检查步骤**：

```bash
# 第 1 步：检查所有容器是否正常运行
docker ps

# 第 2 步：检查后端日志是否有错误
docker logs gbb-backend --tail 30

# 第 3 步：直接测试后端健康检查
curl -s http://localhost:3000/api/v1/health
# 如果连不上，说明后端没启动
# 如果能返回 JSON，说明 nginx 代理有问题

# 第 4 步：检查 nginx 日志
docker logs gbb-nginx --tail 20
```

**常见原因①**：后端 Crash Loop（不断重启）

```
现象：gbb-backend 状态为 Restarting (N)
```

**解决方案**：
```bash
# 查看后端日志找到原因
docker logs gbb-backend --tail 50

# 常见的错误：
# - 缺少数据库表 → 手动创建表（参考 7.4）
# - JWT_SECRET 仍为默认值 → 修改 .env
# - 数据库连接不上 → 检查 postgres 是否 healthy
```

**常见原因②**：数据库表不存在

```
Error: Invalid `prisma.memberLevel.count()`:
The table `public.member_levels` does not exist in the current database.
```

**解决方案**：手动创建缺少的表：
```bash
docker exec gbb-postgres psql -U postgres -d baby_photo_db -c "
CREATE TABLE IF NOT EXISTS public.member_levels (
    id SERIAL PRIMARY KEY,
    level INTEGER NOT NULL UNIQUE,
    name VARCHAR(50) NOT NULL,
    min_growth INTEGER NOT NULL,
    max_growth INTEGER NOT NULL,
    benefits JSONB,
    icon VARCHAR(255),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
"
```

**常见原因③**：健康检查命令格式错误

nginx 显示 `unhealthy` 或 `Restarting`，检查健康检查日志：

```bash
docker inspect --format='{{json .State.Health}}' gbb-backend | jq .
# 如果看到 "Prepended http:// to '||'" 说明格式错误
```

**错误格式**（CMD exec 格式会把 `||` 当作 wget 参数）：
```yaml
test: ["CMD", "wget", "--no-verbose", "--tries=1", "--spider", "http://localhost:3000/api/v1/health", "||", "exit", "1"]
```

**正确格式**（CMD-SHELL 格式，整个命令由 shell 解释）：
```yaml
test: ["CMD-SHELL", "wget --no-verbose --tries=1 --spider http://localhost:3000/api/v1/health || exit 1"]
```

**常见原因④**：后端只监听 127.0.0.1，nginx 访问不到

```bash
# 检查后端日志
docker logs gbb-backend | grep 监听
# 如果显示：监听地址: 127.0.0.1 (仅本地)
# 需要改为 0.0.0.0
```

解决方法：修改 `server/baby-photo-backend/src/main.ts`，将监听地址改为 `0.0.0.0`：
```typescript
const listenHost = '0.0.0.0';  // Docker 环境必须监听所有网络接口
```

> **原因解释**：`127.0.0.1` 是"本地回环地址"，每个容器都有自己的 127.0.0.1，容器 A 访问 127.0.0.1 只能到容器 A 自己，到不了容器 B。而 `0.0.0.0` 是"所有网络接口"，监听此地址后其他容器可以通过 Docker 网络 IP 访问。nginx 在另一个容器中，所以后端必须监听 `0.0.0.0`。

### 10.3 构建时内存不足（OOM）

**现象**：执行 `docker compose build` 时，输出包含：

```
FATAL ERROR: Ineffective mark-compacts near heap limit Allocation failed
JavaScript heap out of memory
```

**原因**：RackNerd VPS 只有 2GB 内存，Node.js 默认堆内存上限约 1.4GB，编译大型 TypeScript 项目时不够用。

**解决方案**（已在 Dockerfile 中添加）：
```dockerfile
ENV NODE_OPTIONS="--max-old-space-size=2048"
```
这个环境变量将 Node.js 堆内存上限提高到 2GB，充分利用服务器内存。

## 11. 附录：Docker 入门

### 11.1 什么是 Docker？

Docker 是一种"容器"技术，可以理解为一个轻量级的虚拟机。

- **传统方式**：在服务器上直接安装 Node.js、PostgreSQL、Redis、nginx... 每个都要配置、版本冲突、卸载麻烦
- **Docker 方式**：每个应用打包在自己的"容器"里，互不干扰，一条命令启动所有服务

类比：
- **镜像（Image）** = 安装包 / ISO 文件
- **容器（Container）** = 安装好并正在运行的程序
- **Docker Compose** = 一键安装脚本（定义多个服务如何配合）

### 11.2 什么是 docker-compose.yml？

`docker-compose.yml` 是一个 YAML 格式的配置文件，描述了这个项目需要哪些服务、每个服务用什么镜像、端口如何映射、环境变量是什么。

以本项目为例，它定义了 6 个服务（PostgreSQL、Redis、后端、前端、nginx、Prisma Studio），每个服务都有各自的配置。执行 `docker compose up -d` 时，Docker 会按照这个文件创建并运行所有容器。

### 11.3 什么是 Dockerfile？

Dockerfile 是构建镜像的"配方"。它告诉 Docker 如何一步步构建一个可运行的镜像。

本项目的后端 Dockerfile 使用**多阶段构建**：
- **Stage 1 (base)**：安装所有 npm 依赖
- **Stage 2 (builder)**：编译 TypeScript 代码
- **Stage 3 (runner)**：只复制编译好的 JS 文件，生成精简的生产镜像

这样最终镜像只包含运行所需的文件，不包含源码和构建工具，更小、更安全。

### 11.4 常用 Docker 命令速查

| 命令 | 作用 | 类比 |
|------|------|------|
| `docker ps` | 查看运行中的容器 | `tasklist` 查看正在运行的进程 |
| `docker logs 容器名` | 查看日志 | 查看程序的输出 |
| `docker stop 容器名` | 停止容器 | 关闭程序 |
| `docker rm 容器名` | 删除容器 | 卸载程序 |
| `docker restart 容器名` | 重启容器 | 重新启动程序 |
| `docker compose up -d` | 启动所有服务 | 一键启动整套系统 |
| `docker compose down` | 停止所有服务 | 一键关闭整套系统 |
| `docker compose build` | 构建镜像 | 编译安装包 |
| `docker exec -it 容器名 /bin/sh` | 进入容器内部 | SSH 登录到虚拟机 |
