微信小程序积分 + 成本利润统计 完整方案                    

 Context

 用户微信小程序已有积分页面（pages/member/points/）但使用模拟数据。后端 /wx-points/my API 不存在。WxUser 模型已有 growthPoints
 字段但从未写入。同时，Product 模型有 costPrice 字段但从未关联到订单，导致无法计算利润。

 扩展需求（用户明确要求）：
 1. 积分兑换比例：100积分=1分钱(¥0.01)，10000积分=1元钱(¥1)，即 1积分=¥0.0001
 2. 积分可在结算时使用抵扣
 3. 加入成本利润统计，显示实际收入和利润

 Phase 1: 数据库 Schema 变更

 修改 prisma/schema.prisma

 1.1 新增 PointsTransaction 模型（在 WxUser 之后）

 model PointsTransaction {
   id          String   @id @default(cuid())
   wxUserId    String   @map("wx_user_id")
   type        String   // EARN / SPEND / EXPIRE / ADMIN
   amount      Int      // 正=获得, 负=消耗
   balance     Int      // 交易后余额
   source      String?  // ORDER / CHECKIN / REDEEM / ADMIN
   sourceId    String?  // 关联单号(order_no)
   description String?  @db.Text
   createdAt   DateTime @default(now()) @map("created_at")
   wxUser      WxUser   @relation(fields: [wxUserId], references: [id])
   @@index([wxUserId, createdAt])
   @@map("points_transactions")
 }

 1.2 Order 模型新增字段（在 discountAmount 之后）

   pointsUsed          Int      @default(0) @map("points_used")           // 使用的积分数量
   pointsDeductAmount  Decimal  @default(0) @map("points_deduct_amount") @db.Decimal(10, 2) // 积分抵扣金额
   totalCostAmount     Decimal  @default(0) @map("total_cost_amount") @db.Decimal(12, 2)    // 总成本
   profitAmount        Decimal  @default(0) @map("profit_amount") @db.Decimal(12, 2)       // 利润 = paidAmount - totalCostAmount

 1.3 OrderItem 模型新增字段（在 totalPrice 之后）

   costPrice   Decimal  @default(0) @map("cost_price") @db.Decimal(10, 2)  // 创建订单时的成本价快照

 Phase 3: 后端 — 订单创建时积分抵扣 + 成本记录

 3.1 修改 create-wx-order.dto.ts

 新增 @IsOptional() @IsInt() @Min(0) pointsUsed?: number。

 3.2 修改 wx-order.service.ts createOrderFromCart() 方法

 在"5. 应用优惠券"步骤之后，新增"5b. 积分抵扣"逻辑：

 1. 如果 dto.pointsUsed > 0:
    - 查询 WxUser.growthPoints，验证余额充足
    - 计算 pointsDeductAmount = Math.floor(pointsUsed * 0.0001 * 100) / 100
      保留2位小数（例如: 1500积分 × 0.0001 = ¥0.15）
    - 保证 pointsDeductAmount ≤ finalAmount（抵扣不超订单总额）
    - finalAmount 减去 pointsDeductAmount
    - 在事务中:
      a. 创建 PointsTransaction (type=SPEND, source=REDEEM, amount=-pointsUsed)
      b. 更新 WxUser.growthPoints -= pointsUsed
      c. 创建订单时写入 pointsUsed / pointsDeductAmount

 同时在创建 OrderItem 时，对 PRODUCT 类型的 item 从 Product.costPrice 读取成本并写入 OrderItem.costPrice，并累加 totalCostAmount 到
 Order。

 3.3 计算并存储利润

 订单创建时：profitAmount = 0（支付后才算真正的利润）。
 在支付回调中更新利润（见 Phase 4）。

 Phase 4: 后端 — 支付成功后积分发放 + 利润更新

 修改 payments.service.ts handlePaymentResult()

 在 FULLY_PAID 分支中（第1290-1306行），updateOrderAfterPayment 之后新增：

 4.1 积分发放 awardOrderPoints(prisma, order, wxUserId)
 - 只在 wxUserId 不为空时执行
 - 积分 = Math.floor(Number(order.totalAmount)) × 1 + 10（1元=1基础分 + 10奖励分）
 - 使用 Prisma $transaction：
   - 读取当前 WxUser.growthPoints
   - 创建 PointsTransaction(type=EARN, source=ORDER, amount=+积分)
   - 更新 WxUser.growthPoints += 积分

 幂等性保证：handlePaymentResult 本身幂等，通过 transactionId 防重复。

 4.2 利润计算 updateOrderProfit(prisma, order)
 - 查询所有 OrderItem，求和 costPrice × quantity → totalCostAmount
 - profitAmount = paidAmount - totalCostAmount
 - 更新 Order 的 totalCostAmount 和 profitAmount
