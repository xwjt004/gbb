## 2. 服务器初始化

### 2.1 首次连接服务器

```bash
ssh ubuntu@81.68.236.12
```

### 2.2 配置 SSH 密钥登录（推荐）

```bash
# 生成 SSH 密钥对（如果还没有）
ssh-keygen -t ed25519 -C "your-email@example.com"

# 将公钥复制到服务器
ssh-copy-id ubuntu@81.68.236.12
```

为了方便后续操作，可以在本地 `~/.ssh/config` 中添加服务器别名：

```bash
Host gbb-server
  HostName 81.68.236.12
  User ubuntu
  IdentityFile ~/.ssh/id_ed25519
```

之后就可以用 `ssh gbb-server` 快速连接了。

### 2.3 安装 Docker

```bash
# 使用腾讯云镜像源加速
curl -fsSL https://mirrors.tencent.com/docker-ce/linux/ubuntu/gpg | sudo apt-key add -
sudo add-apt-repository "deb [arch=amd64] https://mirrors.tencent.com/docker-ce/linux/ubuntu $(lsb_release -cs) stable"
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 将当前用户加入 docker 组
sudo usermod -aG docker $USER
```

### 2.4 配置 Docker 镜像加速

```bash
sudo mkdir -p /etc/docker
sudo tee /etc/docker/daemon.json << 'EOF'
{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com",
    "https://docker.m.daocloud.io"
  ]
}
EOF
sudo systemctl daemon-reload
sudo systemctl restart docker
```

## 3. 部署项目代码

### 3.1 创建项目目录

```bash
mkdir -p /opt/gbb
cd /opt/gbb
```

### 3.2 获取代码

从 Git 仓库克隆（推荐）：

```bash
git clone <你的仓库地址> .
```

或从本地打包上传：

```bash
# 在本地打包项目（排除不需要的文件）
tar czf gbb.tar.gz \
  --exclude='node_modules' \
  --exclude='dist' \
  --exclude='.git' \
  --exclude='.env' \
  -C /path/to/project .

# 上传到服务器
scp gbb.tar.gz ubuntu@81.68.236.12:/opt/gbb/

# 在服务器解压
cd /opt/gbb && tar xzf gbb.tar.gz
```

### 3.3 创建项目所需目录

```bash

sudo mkdir -p /opt/gbb/www/admin        # 前端静态文件
sudo mkdir -p /opt/gbb/uploads          # 上传文件
sudo mkdir -p /opt/gbb/certs            # 微信支付证书
sudo mkdir -p /opt/gbb/backups          # 备份目录
```

## 5. SSL/HTTPS 证书

### 5.1 申请 Let's Encrypt 免费证书

```bash
# 安装 Certbot
sudo apt install -y certbot

# 申请证书
sudo certbot certonly --standalone -d guaibaobao.cn -d www.guaibaobao.cn

# 复制到 nginx SSL 目录
sudo cp /etc/letsencrypt/live/guaibaobao.cn/fullchain.pem /opt/gbb/nginx/ssl/
sudo cp /etc/letsencrypt/live/guaibaobao.cn/privkey.pem /opt/gbb/nginx/ssl/
sudo chmod 644 /opt/gbb/nginx/ssl/*.pem
```

### 5.2 设置自动续期

```bash
sudo crontab -e
# 添加：
0 3 * * * certbot renew --quiet --post-hook "cp /etc/letsencrypt/live/guaibaobao.cn/fullchain.pem /opt/gbb/nginx/ssl/ && cp /etc/letsencrypt/live/guaibaobao.cn/privkey.pem /opt/gbb/nginx/ssl/ && docker exec gbb-nginx nginx -s reload"
```

## 6. 构建与启动

### 6.1 生产环境架构

本项目的生产部署使用以下架构：

```
用户请求 → nginx（80/443）
              ├── /api/v1/* → 反向代理到 gbb-backend:3000
              ├── /uploads/* → 反向代理到 gbb-backend:3000
              ├── /admin/*   → 静态文件（/opt/gbb/www/admin/）
              └── /          → 静态文件（/opt/gbb/www/）
```

| 服务 | 容器名 | 部署方式 |
|------|--------|---------|
| PostgreSQL | gbb-postgres | Docker 容器（官方镜像 postgres:17-alpine） |
| Redis | gbb-redis | Docker 容器（官方镜像 redis:7-alpine） |
| 后端 API | gbb-backend | Docker 容器（自定义镜像 baby-photo-backend:latest） |
| Nginx | gbb-nginx | Docker 容器（nginx:alpine，配置挂载） |

### 6.2 后端部署

生产环境使用自定义 Docker 镜像 `baby-photo-backend:latest`，后端 dist 通过卷挂载方式同步。

#### 首次部署

```bash
cd /opt/gbb

# 启动数据库
docker compose up -d postgres redis

# 构建后端镜像
docker build -t baby-photo-backend:latest ./server/baby-photo-backend

# 启动后端
docker compose up -d backend
```

#### 后端代码更新（仅代码变更，无 Schema 变更）

后端采用 dist 挂载方式，TypeScript 编译后的 JS 文件从宿主机 `/opt/gbb/backend-dist/` 挂载到容器 `/app/dist`，**无需重建 Docker 镜像**：

```bash
# 在本地编译 TypeScript
cd server/baby-photo-backend
npx tsc

# 上传 dist 到服务器
tar czf - -C dist . | ssh ubuntu@81.68.236.12 "sudo tar xzf - -C /opt/gbb/backend-dist && sudo chown -R ubuntu:ubuntu /opt/gbb/backend-dist"

# 重启容器
ssh ubuntu@81.68.236.12 "docker restart gbb-backend"
```

#### 后端代码更新（含 Prisma Schema 变更）

如果修改了数据模型，需要重建 Docker 镜像（Prisma Client 内置在镜像中）：

```bash
# 在本地构建 Docker 镜像
cd server/baby-photo-backend
docker build -t baby-photo-backend:latest .

# 流式传输到服务器并加载
docker save baby-photo-backend:latest | ssh ubuntu@81.68.236.12 "docker load"

# 上传最新 dist
npx tsc
tar czf - -C dist . | ssh ubuntu@81.68.236.12 "sudo tar xzf - -C /opt/gbb/backend-dist && sudo chown -R ubuntu:ubuntu /opt/gbb/backend-dist"

# 重启容器
ssh ubuntu@81.68.236.12 "docker stop gbb-backend && docker rm gbb-backend && cd /opt/gbb && docker compose up -d backend"
```

### 6.3 前端部署（管理后台）

前端在本地构建后上传到服务器，nginx 直接托管静态文件：

```bash
# 在本地执行
cd client/admin-frontend

# 构建（base 路径为 /admin/）
npm run build

# 上传到服务器
tar czf - dist/ | ssh ubuntu@81.68.236.12 "sudo tar xzf - -C /opt/gbb/www/admin && sudo chown -R ubuntu:ubuntu /opt/gbb/www/admin"
```

注意：管理后台构建配置了 `base: '/admin/'`，路由使用 `basename="/admin"`，所有资源路径以 `/admin/` 开头。

### 6.4 验证部署

```bash

docker ps

# 测试后端健康检查
curl https://guaibaobao.cn/api/v1/health
# 期望输出：{"status":"ok","services":{"database":"connected","redis":"connected"},...}

# 测试管理后台
curl -sI https://guaibaobao.cn/admin/ | head -3
# 期望输出：HTTP/2 200
```

## 7. 数据库迁移

### 7.1 自动迁移

后端容器启动时，`docker-entrypoint.sh` 会自动执行数据库迁移：

```bash
# 手动执行迁移
docker exec gbb-backend sh -c "cd /app && npx prisma migrate deploy"
```

### 7.2 迁移故障处理

如果迁移 `P3018` 错误（列已存在等），标记为已应用：

```bash
docker exec gbb-backend sh -c "cd /app && npx prisma migrate resolve --applied <迁移名称>"
```

### 7.3 数据库备份

```bash
TIMESTAMP=$(date +%Y%m%d_%H%M%S)
docker exec gbb-postgres pg_dump -U baby_photo -d baby_photo \
  --create --clean --if-exists \
  --no-owner --no-privileges | gzip > \
  /opt/gbb/backups/db-backup-$TIMESTAMP.sql.gz
```

## 8. 常见问题排查

### 8.1 后端 Crash Loop

```bash
docker logs gbb-backend --tail 50
```

常见原因：
- **`Cannot find module '/app/dist/main.js'`**：dist 上传路径错误。检查 `/opt/gbb/backend-dist/` 目录结构应为扁平（文件直接在目录下，而非 `dist/dist/` 嵌套）
- **数据库连接失败**：检查 postgres 是否 healthy
- **JWT_SECRET 未设置**：检查 .env 文件

### 8.2 nginx 启动失败

```bash
docker logs gbb-nginx
```

常见原因：SSL 证书文件不存在或权限错误。

### 8.3 API 返回 502

```bash
# 重建容器后 nginx 需要重新加载（容器 IP 变更）
docker exec gbb-nginx nginx -s reload
```
