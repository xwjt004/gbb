## 任务清单

### Task 1 — 生产 API 地址配置

**当前问题**: `config/index.ts` 硬编码为 `development`，使用 cpolar 隧道地址。

**涉及文件**: `client/psyd/miniprogram/config/index.ts`

**操作步骤**:

```typescript
// 将 env 从 'development' 改为 'production'
const env = 'production';

// 将 production 的 baseUrl 从占位符改为实际域名
production: {
  baseUrl: 'https://your-domain.com/api/v1',  // → 改为实际域名
}
```

**验证标准**:
- 小程序启动后 console 输出 `BASE_URL: https://实际域名.com/api/v1`
- 所有 `wx.request` 的 URL 开头均为生产域名
- 图片 URL `getImageUrl()` 基于生产域名拼接

### Task 2 — 环境变量凭据配置

**当前问题**: `.env` 中微信凭据全为占位符。

**涉及文件**: `server/baby-photo-backend/.env`

**必须配置**:

| 变量 | 值 | 来源 |
|------|-----|------|
| `WX_APP_ID` | `wx9cc0b9d42349bc3e` | project.config.json (已有) |
| `WX_APP_SECRET` | 实际 AppSecret | 微信公众平台 → 开发→开发管理→开发设置 |

**操作步骤**:
1. 登录 [微信公众平台](https://mp.weixin.qq.com/) → `wx9cc0b9d42349bc3e`
2. 开发管理 → 开发设置 → 复制 AppSecret
3. 更新 `.env`:
   ```
   WX_APP_ID=wx9cc0b9d42349bc3e
   WX_APP_SECRET=<实际值>
   WECHAT_APPID=wx9cc0b9d42349bc3e
   ```

**验证标准**:
- 后端启动后调用 `POST /api/v1/wx-auth/login` 使用真实 code 返回 JWT
- 不报 `微信登录失败: 配置缺失`

### Task 4 — 微信支付真实配置

**当前问题**: `DEV_MODE_MOCK_PAYMENT=true`，所有支付为模拟。

**涉及文件**:
- `client/psyd/miniprogram/pages/payment/payment.ts` (第 5 行)
- `server/baby-photo-backend/.env` (PAYMENT_MOCK_MODE)

**操作步骤**:

1. **微信商户平台配置**:
   - 登录 [pay.weixin.qq.com](https://pay.weixin.qq.com/)
   - 关联 AppID: `wx9cc0b9d42349bc3e`
   - 获取商户号 (MCH_ID)
   - 设置 API 密钥 (APIv2 和 APIv3)
   - 下载并安装商户证书

2. **更新 .env**:
   ```
   WECHAT_MCH_ID=实际商户号
   WECHAT_API_KEY=实际API密钥
   WECHAT_API_V3_KEY=实际APIv3密钥
   WECHAT_SERIAL_NO=实际证书序列号
   WECHAT_PRIVATE_KEY_PATH=./certs/apiclient_key.pem
   WECHAT_CERT_PATH=./certs/apiclient_cert.pem
   WECHAT_NOTIFY_URL=https://your-domain.com/api/v1/payments/wx-notify
   PAYMENT_MOCK_MODE=false
   ```

3. **支付页面关闭模拟**:
   ```typescript
   // payment.ts:5
   const DEV_MODE_MOCK_PAYMENT = false;
   ```

**验证标准**:
- 支付页面调起真实微信支付界面
- 支付成功后异步通知到达 `wx-notify`
- 退款流程正常工作

## 风险与注意事项

1. **支付审核**: 微信支付审核需 1-3 个工作日，建议尽早提交商户资料
2. **小程序审核**: 首次提交审核需 1-7 个工作日，建议预留时间
3. **API 兼容**: 生产环境 `wx.request` 必须使用 HTTPS，域名需在白名单
4. **数据隔离**: 生产数据库与开发数据库隔离，避免开发数据污染
5. **回退方案**: 保留 `PAYMENT_MOCK_MODE=true` 作为支付异常时的回退手段
