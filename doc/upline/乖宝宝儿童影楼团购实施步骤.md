#### 方法一：使用 Prisma Migrate（推荐，如有干净的 migration 历史）

```bash
cd server/baby-photo-backend
# 创建新迁移
npx prisma migrate dev --name add_group_buy_v1.4

```

### 2.3 Prisma Schema 变更（v1.4）

```diff
 model GroupBuyActivity {
-  packageId     Int
+  packageId     Int?
+  productId     Int?
   minCount      Int      @default(3)
+  maxCount      Int?
-  package       Package  @relation(fields: [packageId], references: [id])
+  package       Package? @relation(fields: [packageId], references: [id])
+  product       Product? @relation(fields: [productId], references: [id])

+  @@index([productId])
 }

+model GroupBuyTier {
+  id        Int      @id @default(autoincrement())
+  packageId Int?
+  productId Int?
+  minCount  Int
+  price     Decimal  @db.Decimal(10, 2)
+  createdAt DateTime @default(now())
+  package   Package? @relation(fields: [packageId], references: [id], onDelete: Cascade)
+  product   Product? @relation(fields: [productId], references: [id], onDelete: Cascade)
+
+  @@index([packageId, minCount])
+  @@index([productId, minCount])
+  @@map("group_buy_tiers")
+}
```

## 五、生产部署流程

### 5.1 Docker 部署

```bash
# 1. 构建后端镜像
cd server/baby-photo-backend
docker build -t baby-photo-backend:patched .

# 2. 构建前端镜像
cd client/admin-frontend
docker build -t baby-photo-admin-frontend .

# 3. 启动服务
docker compose -f docker-compose.yml --profile production up -d
```

### 5.2 手动部署（非 Docker）

```bash
# 后端
cd server/baby-photo-backend
npm ci
npx prisma generate
npm run build

# 数据库同步
npx prisma db push --accept-data-loss

# 启动
node dist/main.js

# 前端
cd client/admin-frontend
npm ci
npm run build
```

### 5.3 Nginx 反向代理配置

```
location /api/v1/ {
    proxy_pass http://gbb-backend:3000/api/v1/;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_read_timeout 120s;
}

location /admin/ {
    root /usr/share/nginx/html;
    try_files $uri $uri/ /admin/index.html;
}

location /admin/assets/ {
    root /usr/share/nginx/html;
    expires 1y;
    add_header Cache-Control public;
}
```

### 5.4 健康检查

```bash
curl http://localhost:3000/api/v1/health
# 预期：{"status":"ok","message":"系统运行正常"}
```

## 七、回滚方案

### 7.1 后端回滚

```bash
# 回滚代码
git revert HEAD --no-commit
git commit -m "revert: 回滚团购 v1.4"

# 回滚数据库
npx prisma db push --accept-data-loss
# 或手动执行 SQL 删除新增表和字段
```

### 7.2 前端回滚

```bash
# 恢复旧版本静态文件
cp -r /backup/admin.old/* /usr/share/nginx/html/admin/
```

### 7.3 小程序回滚

在微信小程序管理后台 → 版本管理 → 选择上一版本 → 提交审核 → 发布
