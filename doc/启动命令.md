# 1.  启动命令

## 1. 数据库（PostgreSQL + Redis）

```bash
# 启动所有服务（PostgreSQL + Redis + 后端 + 前端 + Nginx）
cd D:/vscode/GBBV2
docker-compose up -d

# 仅启动数据库和 Redis
docker-compose up -d postgres redis

# 开发环境（含热重载）
docker-compose -f docker-compose.dev.yml up -d

# 查看容器状态
docker ps
```

---

## 2. 后端服务器

```bash
cd D:/vscode/GBBV2/server/baby-photo-backend

# 安装依赖（首次）
npm install

# 生成 Prisma 客户端
npx prisma generate

# 同步数据库表结构
npx prisma db push

# 启动开发模式（热重载）
npm run start:dev

# 启动生产模式
npm run build && npm run start:prod
```

> 服务默认运行在 `http://localhost:3000`

---

## 3. 管理后台

```bash
cd D:/vscode/GBBV2/client/admin-frontend

# 安装依赖（首次）
npm install

# 启动开发模式（热重载）
npm run dev

# 构建生产版本
npm run build
```

> 服务默认运行在 `http://localhost:3001`

## 5. 完整项目启动（从零开始）


## 常用命令速查

| 操作 | 命令 |
|------|------|
| 停止所有容器 | `docker-compose down` |
| 查看后端日志 | `docker-compose logs -f backend` |
| 更新数据库模型 | `npx prisma db push`（在 backend 目录） |
| 查看 Prisma Studio | `npx prisma studio`（在 backend 目录） |
| 运行后端测试 | `npm test`（在 backend 目录） |
| 运行前端测试 | `npm test`（在 admin-frontend 目录） |
| 前端代码检查 | `npm run lint`（在 admin-frontend 目录） |

# 2.  杀死端口命令
务编译启动了，但端口 3000 已被占用（上次启动的进程还在）。先杀掉旧进程：
                                         
  npx kill-port 3000                          
                               
  如果没装 kill-port，用这个：
  netstat -ano | findstr :3000
  taskkill /PID <PID> /  F  
   然后 npm run start:dev 就可以了。
