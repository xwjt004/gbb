# Phase 2: 功能增强与体验优化

## 目标

在基础设施（Phase 1）基础上，聚焦核心业务场景的功能补全和操作体验提升。

---

## 2.1 操作日志 / 审计日志

**现状**: `status_change_logs` 表仅记录订单状态变更，缺少全局操作追踪。

- [x] 创建 `operation_logs` 表（模块、操作类型、操作人、IP、请求详情、耗时）
- [x] 后端：NestJS 全局 interceptor 自动记录增删改操作
- [x] 前端：操作日志查询页面（按模块、操作人、时间筛选）
- [ ] 关键操作埋点：支付确认、退款、库存调整、订单取消、价格修改

---

## 2.2 权限与角色管理

**现状**: 只有 `isAdmin` 字段区分管理员/普通用户，无细分权限。

- [x] Prisma: 新增 `roles`、`role_permissions` 表
- [x] 后端: 角色 CRUD + 权限守卫装饰器 `@RequirePermission('orders:edit')`
- [x] 前端: 角色管理页面（创建角色、勾选权限树）
- [x] 前端: 用户管理页增加角色分配
- [ ] 菜单/按钮级权限控制（无权限时隐藏或禁用）

---

## 2.3 通知发送能力

**现状**: `AppNotification` 只存记录，未对接真实发送渠道。

- [x] 邮件发送：集成 nodemailer，对接 SMTP
- [x] 微信模板消息：桩服务已就绪，接入需配置 WX_APP_ID + 用户订阅模板
- [x] 后台手动触发重试失败通知（单条 + 批量）
- [x] 通知模板管理（可配置邮件/推送模板内容）

---

## 2.4 批量操作增强

**现状**: 各页面缺少批量操作能力。

- [ ] 订单列表：批量确认、批量完成、批量取消
- [ ] 商品列表：批量上架/下架、批量改分类
- [ ] 用户列表：批量导出、批量发券
- [ ] 通用：选中后底部操作栏 + 二次确认弹窗

---

## 2.5 数据导入能力

**现状**: 只有导出，没有导入。

- [ ] 商品导入：Excel 上传 → 预览 → 确认导入（含字段校验）
- [ ] 用户导入：批量导入微信用户或后台用户
- [ ] 时间槽导入：按 Excel 模板批量创建时间槽
- [ ] 通用组件：`FileUpload` + 解析器 + 校验器 + 结果展示

---

## 2.6 搜索与筛选优化

**现状**: 全局搜索功能较基础。

- [ ] 后端：全局搜索统一 Elasticsearch 或 PostgreSQL 全文检索
- [ ] 前端：搜索框增加分类筛选（订单/用户/商品/套餐）
- [ ] 搜索结果高亮关键词
- [ ] 最近搜索记录 + 常用搜索保存

---

## 2.7 性能优化

- [ ] 列表页虚拟滚动（商品/订单/用户等大列表）
- [ ] 后端查询加索引（按实际慢查询分析）
- [ ] 前端路由懒加载检查 + 组件按需加载
- [ ] 图片懒加载 + 压缩策略优化

---

## 优先级建议

| 优先级 | 项目 | 工作量 | 说明 |
|--------|------|--------|------|
| P0 | 权限与角色管理 | 大 | 基础安全设施，影响所有后续开发 |
| P1 | 批量操作增强 | 中 | 日常运营高频需求 |
| P1 | 操作日志 | 中 | 线上问题排查必备 |
| P2 | 数据导入 | 中 | 减少人工录入工作量 |
| P2 | 通知发送能力 | 大 | 需对接外部服务 |
| P3 | 搜索优化 | 中 | 依赖 Elasticsearch 部署 |
| P3 | 性能优化 | 持续 | 随开发逐步进行 |
