# Phase 4: 数据分析与自动化

## 目标

从"记录系统"进化为"数据驱动"，用数据辅助决策，用自动化替代重复人工。

---

## 4.1 经营数据看板

**现状**: 现有 Dashboard 展示基础统计，缺少深度分析。

- [ ] 自定义报表生成器（拖拽选择指标 + 维度 + 时间范围）— deferred, 高阶 BI 功能
- [x] 核心指标趋势图（营收、订单量、客单价、转化率环比/同比）
- [x] 门店经营日报/周报自动生成 + 推送
- [x] 对比分析（本月 vs 上月、同期 vs 目标）
- [x] 数据下钻（从总览 → 分类 → 明细逐层展开）

---

## 4.2 库存智能预测

**现状**: 库存管理依赖人工判断，无预测能力。

- [x] 销量预测（基于历史数据用移动平均/指数平滑预估未来销量）
- [x] 安全库存自动计算（根据采购周期 + 销量波动 + 服务水平）
- [x] 补货建议自动生成（哪些商品需要补货、建议数量）
- [x] 呆滞库存预警（超过 N 天未动销自动标记）
- [x] 库存周转率分析 + 优化建议

---

## 4.3 自动化工作流

**现状**: 关键操作依赖人工触发。

- [x] 订单超时自动取消（确认后 N 小时未付定金自动取消）
- [x] 拍摄完成后自动触发选片提醒（微信通知客户）
- [x] 库存低于阈值自动生成采购建议单
- [x] 定期自动备份 + 备份有效性检查
- [x] 自定义自动化规则引擎（如果条件 A → 执行动作 B）

---

## 4.4 智能营销

**现状**: 无系统化营销能力。

- [x] 客户分群（按消费频次、客单价、偏好标签分组）
- [x] 精准推送（分群客户定向推送优惠券/活动通知）
- [x] 营销效果追踪（推送 → 打开 → 转化 漏斗分析）
- [x] 自动营销规则（如：30 天未消费 → 自动发放回归优惠券）
- [ ] A/B 测试（不同优惠力度/文案效果对比）— deferred

---

## 4.5 数据分析中心

**现状**: 数据分散在各个页面，无统一分析入口。

- [ ] 自助查询（类 SQL 或可视化查询构建器）— deferred, 高阶 BI 功能
- [x] 预置分析模板（营收分析、客户分析、商品分析、摄影分析）
- [ ] 数据导出到外部 BI 工具（CSV/Excel/API 对接）— 已有独立导出模块，待整合
- [ ] 大屏模式（店内展示大屏，实时更新经营数据）— 待规划

---

## 优先级建议

| 优先级 | 项目 | 工作量 | 说明 |
|--------|------|--------|------|
| P0 | 自动化工作流 | 大 | 减少人工操作，降低出错率 |
| P1 | 经营数据看板 | 中 | 管理层决策支持 |
| P2 | 库存智能预测 | 中 | 降低库存成本 |
| P2 | 智能营销 | 大 | 提升客户复购率 |
| P3 | 数据分析中心 | 大 | 高阶分析能力 |
