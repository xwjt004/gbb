# 角色权限使用方法

## 系统默认角色

系统初始化时自动创建 4 个默认角色（标记为"系统角色"不可删除）：

| 角色 | 描述 | 权限范围 |
|------|------|----------|
| **超级管理员** | 系统最高权限，可执行所有操作 | `*:*`（全部权限） |
| **管理员** | 日常运营管理权限 | 订单、用户、支付、套餐、商品、通知 |
| **财务** | 财务相关操作权限 | 订单查看、支付管理、报表统计 |
| **客服** | 客户服务相关权限 | 订单查看/编辑、用户查看/编辑 |

> 超级管理员可以创建自定义角色并精确分配权限。

## 权限管理入口

路径：**左侧菜单 → 系统设置 → 角色管理**（`/system/roles`）

### 功能说明

- **新建角色**：点击右上角"新建角色"按钮，填写名称和描述
- **编辑权限**：在权限树中勾选需要的功能节点
- **编辑角色**：在角色列表点击"编辑"按钮
- **删除角色**：系统角色不可删除，自定义角色可删除

### 权限树说明

权限按模块分组，采用树形结构选择。支持三种通配：

| 格式 | 含义 | 示例 |
|------|------|------|
| `*:*` | 所有模块所有操作 | 超级管理员专用 |
| `orders:*` | 订单模块所有操作 | 查看+创建+编辑+删除+审批+退款 |
| `orders:view` | 订单模块查看权限 | 仅查看订单列表和详情 |

每个权限节点包含：
- **模块**（根节点）：如 `订单管理`、`支付管理`、`系统设置`
- **操作**（叶子节点）：如 `查看`、`创建`、`编辑`、`删除`、`审批`、`退款`

## 分配角色给用户

1. 进入 **员工管理 → 员工列表**
2. 点击目标用户的"编辑"按钮
3. 在"角色"下拉框中选择要分配的角色
4. 点击保存

> 一个用户只能分配一个角色，多个用户可以拥有同一角色（一个"员工"即为一个"用户"）。

## 修改密码

路径：登录后点击**右上角用户头像 → 修改密码**（`/system/change-password`）

步骤：
1. 输入当前密码
2. 输入新密码（不少于6位）
3. 确认新密码
4. 点击"确认修改"

---

## 忘记密码

路径：登录页点击"忘记密码？"链接（`/forgot-password`）

步骤：
1. 输入管理员邮箱
2. 点击"发送重置邮件"
3. 查看邮箱中的重置链接（30分钟内有效）
4. 点击链接跳转到重置页面
5. 设置新密码

> 需要配置 `MAIL_HOST`/`MAIL_USER`/`MAIL_PASS` 环境变量才能发送邮件。如未配置，请直接联系超级管理员重置密码。

---

## 注意事项

1. **系统角色**：超级管理员、管理员、财务、客服标记为系统角色，不允许在界面上删除
2. **权限变更**：角色权限修改后，对应用户需要**重新登录**才能获取新的权限
3. **单角色限制**：一个用户只能分配一个角色，不支持多角色叠加
4. **权限基础设施**：当前权限相关的后端守卫和前端路由过滤正在逐步完善中，目前所有接口对所有已登录用户开放
