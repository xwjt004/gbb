# 团购（拼团）功能设计文档

## 一、数据模型

### 1.1 Package（套餐）— 团购配置字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `groupMinCount` | Int? | 成团所需最少人数 |
| `groupPrice` | Decimal? | 团购价（显示用） |
| `groupBuyDescription` | String? | 团购说明文字 |

### 1.2 GroupBuyActivity（团购活动）— `group_buy_activities`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String (UUID) | 主键 |
| `packageId` | Int | 关联套餐 |
| `creatorUserId` | String | 团长（开团人） |
| `minCount` | Int | 成团人数（默认2） |
| `status` | `ACTIVE / SUCCESS / FAILED` | 活动状态 |
| `expiredAt` | DateTime | 过期时间（创建后48小时） |

关联：`package`, `creator`, `participants[]`, `orders[]`

### 1.3 GroupBuyParticipant（参团记录）— `group_buy_participants`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | String (UUID) | 主键 |
| `activityId` | String | 关联活动 |
| `userId` | String | 参团用户 |
| `status` | `JOINED / SUCCESS / REFUNDED` | 参与状态（REFUNDED 已定义但未使用） |
| `joinedAt` | DateTime | 加入时间 |

### 1.4 Order（订单）— 团购关联字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `groupBuyActivityId` | String? | 关联的团购活动 |

### 2.4 核心业务逻辑（group-buy.service.ts）

#### 开团 (`start`)
- 检查套餐存在、用户无进行中的团购
- 默认 `minCount = 2`
- 过期时间 = 当前 + 48小时
- 开团人自动成为第一个参团者（JOINED）

#### 参团 (`join`)
- 失败/已取消的活动不可参团
- 检查是否已参团（防重复）
- 已达到成团条件的直接设为 SUCCESS
- 参团后检查人数是否达到 `minCount` → 自动成团

#### 成团 (`completeGroupBuy` — 私有方法)
- 更新活动状态 → SUCCESS
- 更新所有参团者状态 → SUCCESS
- 发放 9 折优惠券给每位参团者：
  - 查找或创建 `GROUP_BUY` 类型优惠券模板
  - `discountType: PERCENT`, `discountValue: 10`（9折）
  - 适用该套餐，有效期 30 天
  - 每人限领 1 张

#### 过期处理
- 定时任务每小时执行一次（`group-buy.scheduler.ts`）
- 将已过期的 ACTIVE 活动设为 FAILED

## 三、后台管理前端

### 3.1 套餐编辑（PackageForm.tsx）
- 成团人数（`groupMinCount`）— InputNumber
- 团购价（`groupPrice`）— InputNumber，¥ 前缀
- 团购说明（`groupBuyDescription`）— TextArea
- 海报配置：标题、内容、背景色、图片

### 3.2 团购管理列表（Marketing/GroupBuyList.tsx）
- 路径：`/marketing/group-buy`（营销管理菜单下）
- CRUD 操作：创建、编辑、取消、恢复、删除
- 状态筛选：进行中 / 已成团 / 已失败

### 3.3 团购订单列表（Orders/OrderGroupBuyList.tsx）
- 路径：`/marketing/group-buy` 别名路由（在订单管理下也有入口）
- 显示字段：编号、套餐信息（含团购价/原价对比）、团长、入团人数、已付款、未付款、状态、时间
- 统计卡片：总数、进行中、已成团、已失败
- 详情弹窗

### 3.4 套餐列表团购标识（PackageList.tsx）
- "团购设置" 列：显示红色 "团购 ¥xxx" 标签 + 成团人数
- 未设置团购的套餐显示 "-"
- 商户可一目了然哪些套餐有团购活动

### 3.5 订单列表团购标记（OrderList.tsx）
- "订单信息" 列：订单号旁显示红色 "团购" 徽标（icon: TeamOutlined）
- "套餐信息" 列：根据 `groupBuyActivity.status` 显示团购状态标签（已成团/进行中）
- 商户在订单列表中可直接识别团购订单

### 3.6 订单详情团购信息（OrderDetail.tsx）
- "套餐信息" 卡片中显示 "团购信息" 行
- 展示：团购状态标签（已成团/进行中/已结束）+ 成团人数
- 非团购订单不显示该行

### 3.7 后端数据支持
- `orders.service.ts` 的 `findAll`、`findOne`、`findByOrderNo` 查询均增加了 `groupBuyActivity` 关联（含 id/status/minCount）
- `wx-order.service.ts` 的 `getMyOrders`、`getOrderDetail` 查询也增加了 `groupBuyActivity` 关联
- 前端 `Order` 类型增加了 `groupBuyActivityId?: string` 和 `groupBuyActivity?: { id, status, minCount }`
- `paidCount` = 订单中 `paymentStatus` 为 `FULLY_PAID` 或 `PARTIAL_PAID` 的数量
- `unpaidCount` = 参团总人数 - `paidCount`

## 四、微信小程序端

### 4.1 页面结构

| 页面 | 路径 | 说明 |
|------|------|------|
| 套系详情 | `packages/detail/detail` | 入口，显示"拼团优惠"区域 |
| 开团页 | `group-buy/start/start` | 设置成团人数，开团 |
| 团购详情 | `group-buy/detail/detail` | 状态、倒计时、参团、海报 |
| 我的团购 | `group-buy/list/list` | 历史团购列表 |

### 4.2 完整下单流程

```
套系详情 → 拼团按钮 → 开团页 → 开团 → 团购详情页
                                                ↓
                     参团 ← 扫码/分享 ← 团购详情页 ← 其他人
                      ↓
              分享到朋友圈/保存海报（ShareGate）
                      ↓
                POST /group-buy/:id/join
                      ↓
          booking/date → booking/time → booking/info → booking/confirm
                      ↓
                POST /orders（创建订单，totalAmount = groupPrice）
                      ↓
                /pages/payment/payment（支付）

```

### 4.3 团购价格在订单中的体现

**核心发现：团购价格仅用于前端展示，后端不进行价格验证。**

- 套系详情页和团购详情页显示的 `pkgPrice = pkg.groupPrice || pkg.price`
- 参团后通过 booking 流程下单，booking/confirm 页面读取 `packageInfo.price`（即 `groupPrice`）作为 `totalAmount` 提交到后端
- 后端 `orders.service.ts` 第 141 行使用客户端传入的 `totalAmount`，不验证是否等于团购价
- **wx-order 下单流程**（购物车/直接购买）：使用标准价格，但会通过 `wxUserId + packageId` 自动关联 `groupBuyActivityId`
- **booking 流程**（参团后下单）：使用 `groupPrice`，但**不会**设置 `groupBuyActivityId`（orders.service 无此逻辑）

### 4.4 分享机制
- 参团前必须分享到朋友圈或保存海报（ShareGate 弹窗）
- 团购详情页支持分享给好友（`onShareAppMessage`）
- 朋友圈分享（`onShareTimeline`）和 `wx.shareToTimeline`
- 海报使用 Canvas 2D API 生成，含二维码

## 五、团购价格与订单/退款的关系

### 5.1 团购价格是如何在订单列表中体现的

团购活动列表后台页面通过以下方式计算：

1. **`groupPrice`** — 存储在 `Package` 表的 `group_price` 字段，后台直接读取展示
2. **入团人数** — `_count.participants`（Prisma 关联计数）
3. **已付款 / 未付款** — 通过关联的 `orders[]` 遍历，统计 `paymentStatus` 为 `FULLY_PAID` 或 `PARTIAL_PAID` 的订单数量
4. 订单本身的价格是创建时确定的（booking 流程用 `groupPrice`，wx-order 流程用标准价），订单创建后价格不再与团购价关联

### 5.2 团购退款与正常退款的差异

**当前实现：团购退款与正常退款没有区别。**

- 退款走通用支付退款流程（`payments.service.ts`），不区分是否来自团购
- `GroupBuyParticipant` 模型定义了 `REFUNDED` 状态，但**没有任何代码将其设置为 REFUNDED**
- 团购订单退款后，`GroupBuyParticipant` 仍保持 `SUCCESS` 状态
- 已发放的团购 9 折券不会因退款而撤销

### 5.3 待解决的问题

1. **Booking 流程下单未关联团购活动** — `orders.service.ts` 没有设置 `groupBuyActivityId`，导致通过 booking 流程下单的团购用户订单无法在后台团购列表中统计已付款/未付款
2. **退款后未更新参团状态** — 团购订单退款后 `GroupBuyParticipant.status` 仍为 `SUCCESS`
3. **退款后未回收优惠券** — 退款后团购发放的 9 折券仍然可用
4. **团购价未验证** — 后端完全信任客户端传入的 `totalAmount`，没有验证是否与套餐的 `groupPrice` 一致
5. **价格矛盾** — booking 流程用 `groupPrice` 下单，而 wx-order 流程用标准价下单，同一团购活动的参与者可能支付不同价格

## 六、优惠券生成

### 6.1 成团自动发放

成团时 `completeGroupBuy()` 方法：
- 为每个参团者生成 `GROUP_BUY` 类型优惠券
- 折扣值：10（10% off，即9折）
- 有效期：30天
- 适用范围：仅限该套餐
- 防止重复发放（检查 `UserCoupon` 是否已存在）

### 6.2 优惠券模板（Coupon）

| 字段 | 值 |
|------|-----|
| `couponCode` | `GROUP_{packageId}_{timestamp}` |
| `couponName` | `团购9折券（{套餐名}）` |
| `couponType` | `GROUP_BUY` |
| `discountType` | `PERCENT` |
| `discountValue` | 10 |
| `minAmount` | 0（无门槛） |
| `applicableType` | `PACKAGE` |
